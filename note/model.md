# noteの推しアルゴリズム — 構造化まとめ

> 記事「noteの推しアルゴリズムについて」をベースに構造化

---

## 大原則

**「一次情報・生の体験・私の気づき」を応援する**

noteは「その人にしか書けないこと」を最も重視する。AI生成や他者の情報の再構成よりも、自分自身が体験し、考え、感じたことを言語化したコンテンツが優遇される。

### コンテンツ優先順位

```
人間が書いた一次情報 > AI活用して人間が編集 > AI自動生成 > 大量生成
```

---

## AI執筆ルール（必須）

記事の執筆・ブラッシュアップをAIが行う際、以下のルールを厳守すること。

### 体験談・気づき・感情・他者の反応に関する記述は、必ず筆者に確認する

| 対象 | 具体例 |
|------|--------|
| **体験談** | 「最初は〜だった」「〜を試したが失敗した」など、筆者の過去の行動・出来事 |
| **気づき・学び** | 「〜と実感した」「〜だと気づいた」など、筆者の内省・発見 |
| **感情・実感** | 「〜と感じた」「〜が楽しかった」など、筆者の主観的な感覚 |
| **Before/After** | 導入前後の変化、運用期間中の変化 |
| **他者の反応** | 「相棒が〜と言った」「〜してくれた」など、筆者以外の人物の言動 |

### 確認フロー

```
1. AIが体験談・気づき・感情の記述を追加/変更したい場合
     ↓
2. 必ず AskUserQuestion で筆者に内容を提示し、確認を取る
     ↓
3. 筆者の回答に基づいて記述を確定する（推測での確定は禁止）
```

### 禁止事項

- **体験の捏造禁止**: 筆者が経験していないエピソードをAIが創作してはならない
- **感情の推測禁止**: 「〜と感じたはず」と推測して書いてはならない
- **確認なしの追加禁止**: 体験・気づき・感情に関する新規記述は、筆者の確認なしに追加してはならない

### 確認が不要なケース

以下はAIの判断で追加・編集してよい:

- 文章の構成・順序の変更（内容を変えない範囲）
- 見出し・フォーマットの改善
- 既存の事実情報の整理・再構成
- SEO最適化のための表現調整（事実を変えない範囲）
- コードブロック・表・リストのフォーマット調整

## 推されるコンテンツの6カテゴリ

### 1. 体験・一次情報に関する記事

- 自分が実際に経験したこと、やってみたこと
- 失敗談とそこからの学び
- 具体的なケーススタディ（数字・期間・プロセスを含む）
- 専門知識に基づく独自の分析

### 2. 一次創作

- オリジナルの作品（小説、イラスト、音楽、写真など）
- 自分で企画・制作したもの
- 二次創作よりもオリジナル作品が優遇される

### 3. 煽らない私的な感情の発信

- 個人的な感情・内省の共有
- 「私はこう感じた」「こう思った」という主観的な記録
- 攻撃や断罪を目的としない、穏やかなトーン

### 4. コラボレーション誘発コンテンツ

- 読者に問いかけを投げるコンテンツ
- コメント欄での議論を生むテーマ設定
- 「あなたはどうしていますか？」「こんな方法もあります」など、双方向性のある書き方

### 5. 新しい試みを推奨するコンテンツ

- 新しい挑戦の記録
- 実験的な取り組みのレポート
- 「やったことがないことをやってみた」系

### 6. イベント・アクティビティ関連

- オフラインイベントのレポート
- コミュニティ活動の記録
- ワークショップ・勉強会の参加記

## 構造化テキストの要素（SEO/AI推薦されやすい）

記事の「構造」が整っていると、noteのアルゴリズムだけでなく、Google検索やAIレコメンドにも拾われやすくなる。

| 要素 | 説明 |
|------|------|
| **イシューが明確** | 記事冒頭で「誰の・何の問題を・どう解決するか」が明確 |
| **文章が自己完結** | 外部リンクに飛ばなくても記事単体で価値が完結する |
| **意見と事実の区別** | 「〜と感じた」（意見）と「〜だった」（事実）が書き分けられている |
| **適切な引用** | 他者の情報を引用する場合はソースが明記されている |
| **初心者向け入門コンテンツ** | 専門的な内容でも初心者が理解できる説明がある |

---

## 推されにくいコンテンツ

以下のようなコンテンツはアルゴリズム上で冷遇される。

| 種類 | 具体例 |
|------|--------|
| **攻撃・断罪・不安煽り** | 「〇〇はダメだ」「〇〇を許すな」「知らないとヤバい」 |
| **過剰な一般化** | 「〇〇な人はヤバい」「〇〇する人は全員〇〇」 |
| **他人の再生産** | 他者の記事の焼き直し、まとめただけのコンテンツ |
| **フェイク・誤認誘発** | 事実でない情報、誤解を招く表現 |
| **AIの大量生成** | 人間の編集を経ていないAI出力のそのままの公開 |

## 記事ブラッシュアップ チェックリスト

記事を書く/改善する際に確認すべき項目。

### 一次情報・体験

- [ ] 自分自身の体験に基づいているか
- [ ] 具体的な数字・期間・プロセスが含まれているか
- [ ] 失敗談や試行錯誤のプロセスが書かれているか

### 構造・読みやすさ

- [ ] 冒頭でイシュー（誰の・何の問題を・どう解決するか）が明確か
- [ ] 記事単体で価値が完結するか（外部リンクに依存していないか）
- [ ] 意見と事実が区別されているか
- [ ] 見出しにキーワードが含まれているか

### 「私の気づき」

- [ ] 各セクションに著者の気づき・学び・内省が含まれているか
- [ ] 「開発を通じて気づいたこと」「運用してみて分かったこと」が書かれているか

### コラボレーション誘発

- [ ] 読者への問いかけがあるか
- [ ] コメント欄での議論を促す要素があるか
- [ ] 読者が行動に移せる提案があるか

### 初心者向け

- [ ] 専門的な内容でも、非専門家が理解できる説明があるか
- [ ] 技術を使わなくても応用できるポイントが示されているか

### 継続性

- [ ] 開発プロセスや改善の歴史が含まれているか
- [ ] 時系列の変化（運用期間中の学び）が書かれているか

### NG要素の排除

- [ ] 攻撃的・断定的な表現がないか
- [ ] 過剰な一般化がないか
- [ ] 不安を煽る表現がないか
