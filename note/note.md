# 同棲カップルの"お金どうする問題"を、LINE Botで解決した話

## 1. 同棲の"お金どうする問題"

相棒と同棲を始めたとき、毎日がとにかく楽しかった。一緒にごはんを食べて、休みの日は買い物に行って、たまに旅行の計画を立てて。

でも、1ヶ月もしないうちに「あの壁」にぶつかった。

**お金の管理、どうする問題。**

「今月いくら使った？」「この前の外食、どっちが払ったっけ？」「月末の精算、計算めんどくさいな……」

最初はざっくりでいいかと思っていたけど、時間を重ねるごとに確認作業がストレスになっていく。「あれ、この出費どっちだっけ？」みたいな会話が増えると、なんとなく気まずい雰囲気にもなる。

家計簿アプリは世の中にたくさんある。試しにいくつか入れてみたこともある。でも、二人で続けるのが本当に難しい。

- 片方がアプリをインストールしてくれない
- 入力がめんどくさくて3日で開かなくなる
- 共有機能はあるけど操作がいちいち複雑
- 精算の計算を結局手動でやることになる

最初はタイムツリー（TimeTree）に日々の支出を入力する方式をとっていた。でもこれがまた続かない。入力を忘れる日が出てきて、月末はお互い仕事が忙しく精算が後回しになり、気づいたら次の月になっていることもしばしば。**「管理する仕組み自体がストレスになっている」**——これが根本的な問題だった。

「この問題、技術で解決できないか？」

Webエンジニアとしてのスイッチが入った瞬間だった。

## 2. こんな方に読んでほしい

- 誰かと一緒に暮らしていてお金の精算に困っている人
- 家計簿アプリを入れたけど続かなかった経験がある人
- 「二人でお金の話をするのが気まずい」と感じている人
- エンジニアで日常の困りごとを技術で解決したい人

## 3. 作った動機 — エンジニアが自分ごととして向き合った

普段はWebエンジニアとして働いている。技術で問題を解決することが仕事であり、趣味でもある。

だからこそ、同棲中の相棒との間でお金のことをもっとスマートに管理したいと強く思った。

既存の家計簿アプリに対する不満は明確だった。

- **二人それぞれがアプリを入れる必要がある** — これが最初のハードル
- **共有機能があっても操作が複雑** — 結局片方しか使わなくなる
- **レシートの手入力がめんどう** — 毎回金額と店名を打つのは続かない

一番の問題は「相棒にとってのハードル」だと気づいた。

自分はエンジニアだからどんなアプリでも使いこなせる。でも相棒にとって、新しいアプリを入れて、使い方を覚えて、毎回入力するのはかなりのストレスだ。

**「相棒にとってのハードルをゼロにしたい」**

これが最大のモチベーションになった。

相棒は毎日LINEを使っている。朝起きたらLINEを開いて、仕事中もLINEで連絡して、寝る前もLINEを見ている。

なら、**LINEの中で完結すればいいのでは？**

## 4. なぜLINEなのか

LINE Botを選んだ理由はシンプル。

**相棒が既に日常的に使っているサービスだから。**

これに尽きる。

わざわざ別のプラットフォームを介さず、普段の延長線上で使える。「新しいアプリ入れて」と頼む必要がない。これが継続のカギだと確信していた。

導入方法も超シンプルで、グループトークにBotを追加するだけ。LINEのグループを作って、そこにBotを招待する。それだけで二人で家計管理ができるようになる。

通知もLINEで届くから見逃さない。月末のレポートも、毎朝の予定通知も、全部LINEに届く。別のアプリを開く必要は一切ない。

## 5. 二人で決めていた精算ルール

これはBotを作るために新しく決めたルールではない。同棲を始めた頃に二人で「お金の管理どうする？」と話し合って決めたもので、Bot開発よりずっと前からある。

もともとこのルールで運用していたからこそ、そのままBotに組み込むことができた。つまり、**先にルールがあって、後からBotがそれを自動化した**という順番になる。

### 半月担当制 — 「今日どっちが払う？」をなくす

| 期間 | 担当 | 役割 |
|------|------|------|
| 月の前半（1日〜15日） | Aさん | 外食の支払い担当 |
| 月の後半（16日〜月末） | Bさん | 外食の支払い担当 |

担当期間中は、その人が外食費をすべて管理する。「今日どっちが出す？」のやり取りが完全にゼロになる。

### 予算制

- 外食費用は **一人あたり月X万円の予算**（金額は変更可能）
- 担当者は自分の担当期間中、**予算内に収まるように管理する責任**を持つ
- **予算を超過した場合、超過分は担当者が自己負担する**

つまり、担当期間中に使いすぎたら自分の財布が痛む。だからこそ自然と「今月はちょっとセーブしよう」という意識が芽生える。ペナルティではなく、**自分ごととして管理する仕組み**になっている。

### 精算の期限

> **精算は翌月15日まで。それを過ぎたら無効。**

買い物費用・旅行費用は月末に精算額を計算して、実際の送金は各自で行う。その期限が翌月15日まで。期限を決めることで「来月やろう」「再来月でいいか」とズルズル先延ばしにならない。このルールがあるおかげで、お金のことで感情的になることがほぼなくなった。

### まとめると

| ルール | 内容 |
|--------|------|
| 🍽️ 外食の支払い | 半月ごとに担当者を交代 |
| 💰 外食の予算 | 一人月5万円。超過分は担当者負担 |
| 🛒 買い物・旅行 | 各自が払って月末に差額を精算 |
| ⏰ 精算の期限 | 翌月15日まで。過ぎたら無効 |

## 6. どんなものを作ったのか —「家計ぼっと」の全体像

名前は**「家計ぼっと」**。LINEのグループトークに住んでいるBot。

できることを一言で言うと、**LINEで画像や対話形式を行うだけで収支を管理できる**。

### 4つの支出カテゴリー

暮らしのお金を4つのカテゴリーで網羅している。

| カテゴリー | 内容 |
|-----------|------|
| 🍽️ 外食費用 | レストラン・飲食店での食事。一人ずつの予算制 |
| 🛒 買い物費用 | スーパー・コンビニでの買い物。月末精算 |
| ✈️ 旅行費用 | 交通費・宿泊費など旅行の支出。月末精算 |
| 🔄 サブスク | 家賃・Netflix・定期購買など固定費。月初に自動登録 |

### できること一覧

| 機能 | 内容 |
|------|------|
| 📸 レシート画像で自動登録 | レシートやPayPay・Uber Eats等のスクショを送るだけでAIが解析・登録 |
| ✏️ テキストで手入力登録 | レシートがないときは対話形式やワンライナーで手動登録 |
| 🔍 履歴・集計の確認 | 直近の支出一覧や、月別のカテゴリー別集計を確認 |
| 🗑️ 削除・修正 | 間違えて登録した支出を対話形式で取り消し |
| 💰 外食残高の管理 | 予算制で「あといくら使えるか」をリアルタイムに把握 |
| 📊 中間・月末レポート | 15日に中間レポート、月末に精算額・貯金額をまとめた最終レポートを自動送信 |
| 📅 予定リマインド | Googleカレンダーに予定を登録し、当日の朝7:00にLINEで通知 |
| 🔄 サブスク自動登録 | Netflix・家賃・定期購買などを月初に自動で記録 |
| 📆 Googleカレンダー連携 | 支出がカレンダー上に色分けで表示。双方向同期でカレンダー側の編集も反映 |

レシート画像を送って自動登録するのがメインの使い方だが、レシートがない場面でも手入力で登録できるし、中間・月末のレポートで振り返りも自動化されている。予定リマインドやサブスク自動登録など、**家計管理に必要なことがLINEの中ですべて完結する**のがこのBotの特徴。

## 7. 基本の使い方 — レシートを撮って送るだけ

一番よく使う機能は、レシートやPayPayの画面をスクショしてLINEに送信すること。本当にこれだけ。

送信すると、AIが自動で以下の情報を解析してくれる。

- **日付** — レシートに書かれている日付
- **金額** — 支払った金額
- **カテゴリー** — 外食/買い物/旅行を自動判定
- **お店の名前** — どこで使ったか

登録が終わると、Botが即座に返答してくれる。

```
✅ 登録しました！
📝 外食費用: ￥1,280（田中）
🏪 サイゼリヤ
📅 2024/12/03

💰 田中の外食残高: ￥48,720
```

外食費用の場合は残高も一緒に返ってくるので、「今月あといくら使えるか」がすぐわかる。

Googleカレンダーにも自動で登録される。色分けがされていて、外食はフラミンゴ（ピンク系）、買い物はバナナ（黄色系）、旅行はラベンダー（紫系）で表示される。

対応しているのはレシートだけじゃない。**PayPay・LINE Pay・Uber Eats・クレカの明細画面のスクショ**でもOK。電子決済アプリの支払い完了画面を撮って送れば、同じように自動解析してくれる。

## 8. お金の管理ルール

今回作成したBotには、**二人がお金のことでストレスを感じない仕組み**がいくつか組み込まれている。

### 外食費用: 予算制で「使いすぎ」が一目瞭然

外食費用は一人月5万円の予算制（金額は変更できる）。

使うたびに残高が減っていくので、まるでゲームのHP管理みたいな感覚になる。残高が少なくなると自然と「今月はセーブしよう」と意識が変わるし、月末に残高が多いと「今月は節約できた」とポジティブな話題になる。

```
💰 外食残高

田中: ￥35,000
鈴木: ￥42,000

📅 今月の予算: ￥50,000/人
```

### 半月担当制:「今日どっちが払う？」がゼロになる

これが地味に一番効いている仕組みで、お互いにフェアな形式を取れるようになったと実感している。

- 月の前半（1日〜15日）はAさんが外食の支払い担当
- 月の後半（16日〜月末）はBさんが外食の支払い担当

たったこれだけで、レジ前での「どっちが出す？」問題が完全になくなった。「今日は自分の担当期間だから自分が払う」。

### 買い物・旅行: 各自が払って月末に自動精算

買い物や旅行の費用は、普段はそれぞれ気にせず支払いOK。月末にBotが自動で差額を計算してくれる。

```
【🛒 買い物費用 - 精算】
田中 合計: ￥25,000
鈴木 合計: ￥28,000
差額: ￥3,000

→ 田中が鈴木に ￥1,500 を返金してください🙏
```

「今月どっちがいくら払ったか」を自分で計算する必要は一切ない。

### 月末の自動レポート

15日に中間レポート、月末に最終レポートが自動で届く。

最終レポートには、外食の貯金額と買い物・旅行の精算額がまとめて表示される。二人でレポートを見ながら「今月はこうだったね」と振り返るのがちょっとした楽しみになった。

```
━━━━━━━━━━━━━━━━━━━━
📅 月間サマリー
━━━━━━━━━━━━━━━━━━━━

【💰 外食費用 - 貯金額】
田中: ￥25,000 使用 → 貯金成功 ￥25,000 🎉
鈴木: ￥21,000 使用 → 貯金成功 ￥29,000 🎉

【🛒 買い物費用 - 精算】
田中 合計: ￥25,000
鈴木 合計: ￥28,000
差額: ￥3,000

→ 田中が鈴木に ￥1,500 を返金してください🙏
```

## 9. まだまだある便利機能

### テキストで手動登録（@追加）

レシートがないときは、対話形式で1つずつ入力できる。Botが「カテゴリーは？」「金額は？」と順番に聞いてくるから、答えていくだけ。

急いでいるときは、ワンライナーでもOK。

```
@追加 @自分 外食費用 1280
```

`@自分`と入力すると、送信者の名前が自動で入る。いちいちフルネームを打つ必要はない。

### 支出の確認（@履歴・@集計）

`@履歴`で直近の支出を一覧表示。`@集計`で今月のカテゴリー別集計を確認できる。

```
📋 直近の支出履歴

1. 12/03 [外食費用] 田中 ￥1,280
2. 12/02 [買い物費用] 鈴木 ￥2,500
3. 12/01 [外食費用] 田中 ￥980
...
```

過去の月も`@集計 2024/12`のように年月を指定すれば確認できる。

### 予定管理（@予定）

```
@予定 ディナー 12/25 19:00
```

家計簿だけじゃなく、二人のスケジュール管理にも使える。Googleカレンダーに予定が登録されて、当日の朝7:00にLINEで通知が届く。

### サブスク自動登録（@サブスク追加）

Netflix、Spotifyなど毎月固定の支払いを登録しておくと、月初めに自動で「買い物費用」として記録してくれる。週単位の定期便（Amazon定期おトク便など）にも対応している。

```
🔄 サブスク自動登録完了
━━━━━━━━━━━━━━━

2024年12月分のサブスクを登録しました

✅ Netflix（鈴木・¥1,490・12/15）
✅ Amazon定期便 猫砂（田中・¥1,980・12/10）
```

### 家賃自動登録（@家賃追加）

家賃は月末にGoogleカレンダーへ自動登録される。一度設定すれば、毎月の家賃を忘れることなく記録できる。

### 毎朝の予定通知

毎朝7:00にその日の予定をLINEで通知してくれる。朝起きてLINEを開くと、今日の予定が見える。

```
📅 本日の予定 (12/15)

1. 👤 田中
   📝 会議

2. 👤 鈴木
   📝 買い物
```

### 削除・修正も簡単

間違えて登録しても`@削除`で対話形式で取り消せる。

```
🗑️ 以下の支出を削除しました

📅 12/03
📝 外食費用: ￥1,280（田中）
🏪 サイゼリヤ

💰 田中の外食残高: ￥36,280（+￥1,280）
```

外食費用を削除した場合は、残高もちゃんと戻る。

## 10. Googleカレンダーとの連携 — 支出が「見える」ようになる

個人的にお気に入りの機能が、Googleカレンダーとの連携。

支出がカレンダー上に色分けで並ぶので、「今月いつ何に使ったか」が一目瞭然。

| カテゴリー | カレンダーの色 |
|-----------|-------------|
| 🍽️ 外食費用 | フラミンゴ（ピンク系） |
| 🛒 買い物費用 | バナナ（黄色系） |
| ✈️ 旅行費用 | ラベンダー（紫系） |
| 🔄 サブスク | バジル（緑） |

カレンダー上で直接イベントを編集・削除することもできる。毎日深夜3:00にGoogleカレンダーとBotのデータベース間で同期が走るので、LINEから登録した支出はカレンダーに反映され、カレンダー側で手動編集・削除した内容もBotのデータに自動で反映される。

月末にカレンダーを見返すと、お金の使い方のパターンが見えてくる。「先月は旅行多かったな」「今月は外食控えめだった」が視覚的にわかるのは、数字だけのレポートとは違う良さがある。

## 11. 使ってみてどうだったか — リアルな感想

### 導入の瞬間

相棒に「LINEグループにBot入れたから、レシート送ってみて」と言うだけで始められた。

新しいアプリのインストールもアカウント登録も一切なし。相棒もすぐに使い始めてくれた。このハードルの低さが一番大きかったと思う。

[画像: 相棒がレシートを送信して、Botが即座に返答している実際のやり取り]

### 続いている理由

- **レシートを撮って送るだけ。本当にそれだけ。** 手入力がほぼない
- **残高が減っていくのが見えるので、自然と意識が変わる。** 「今月あといくら使える」が常にわかる
- **月末のレポートを二人で見るのが楽しみになった。** お金の振り返りがポジティブな時間になった

### 変わったこと

- 「今月外食多いな」「買い物はいい感じ」という会話が自然に増えた
- **お金の話がストレスではなく、ポジティブな話題に変わった**
- 精算の計算を手動でやる必要がなくなった
- 「どっちが払う？」の問題が半月担当制で完全に解消された

### 月額コスト

運用コストは**約50円/月**。ほぼ無料で動いている。技術選定の話は最後のセクションで詳しく書くけど、サーバーレス構成のおかげでほとんどお金がかからない。

## 12. おわりに — 技術は生活を楽にするためにある

Webエンジニアとしての特権がある。日常の困りごとを、自分の手で解決できること。

大きなサービスを作ることだけが技術ではない。身近な人の「めんどくさい」を解決する小さなツールにも、確かな価値がある。

「家計ぼっと」は、自分と相棒のために作った、二人だけの家計管理ツール。でも、同棲カップル、夫婦、シェアハウスなど、複数人でお金を管理する場面はたくさんある。

この記事が、同じような悩みを持っている人の参考になれば嬉しい。

ソースコードは公開しているので、興味がある方はぜひ覗いてみてほしい。

```
│                                                   │
│   Firestore (NoSQL データベース)                  │
│   ├── users / expenses / settings                 │
│   ├── conversations / subscriptions / rents       │
│                                                   │
│   Cloud Scheduler (6ジョブ: 定期実行)             │
│                                                   │
│   Gemini API (AI画像解析)                         │
│                                                   │
└─────────────────────────────────────────────────┘
                        │
                        ▼
              Google Calendar API
            (支出・予定イベント登録)
```

LINEでレシートを送信すると、Cloud Functionsがwebhookで受け取り、Gemini APIで画像を解析。抽出された支出情報をFirestoreに保存しつつ、Googleカレンダーにもイベントとして登録する。

定期実行はCloud Schedulerが6つのジョブで管理している。

### 13.3 なぜGCPを選んだか

- **サーバーレスで管理不要** — サーバーの面倒を見なくていい
- **無料枠が充実** — Cloud Functions + Firestore + Cloud Scheduler がほぼ無料枠内で収まる
- **Google Calendar APIとの親和性** — 同じGoogleエコシステム内でスムーズに連携できる

### 13.4 なぜFirestoreを選んだか（Cloud SQLとの比較）

データベースの選定で一番悩んだのがここ。Cloud SQL（MySQL/PostgreSQL）も候補に挙がったが、最終的にFirestoreを選んだ。

| 観点 | Firestore | Cloud SQL |
|------|-----------|-----------|
| 月額コスト | $0（無料枠内） | $15〜20（最小構成でも） |
| インスタンス管理 | 不要（完全サーバーレス） | 必要（常時起動） |
| Cloud Functions連携 | ネイティブ対応 | VPC Connector必要（追加コスト） |
| スケーリング | 自動 | 手動設定が必要 |

**年間のコスト差は歴然で、Firestoreなら$0、Cloud SQLは$180〜240（約¥27,000〜36,000）。** 個人プロジェクトにはFirestoreが圧倒的にコスト優位だった。

本システムのデータ構造は「各支出が独立したドキュメント」で、複雑なJOINも不要。Firestoreのドキュメント型データベースと非常に相性が良かった。

### 13.7 技術的に工夫したポイント

**Gemini APIのプロンプト設計**

レシートだけでなく、PayPay・Uber Eats・クレカ明細など多様な支払い画面に対応するため、プロンプトに詳細なカテゴリー判定ルールを記載した。

例えば、「ファミリーマート→買い物費用」「サイゼリヤ→外食費用」「JR東日本→旅行費用」のように、店舗名からカテゴリーを自動判定するルールを組み込んでいる。テイクアウト専門店（丼丸、ほっともっとなど）は外食ではなく買い物費用に分類するなど、実際の使い方に沿った細かいルール設計をしている。

**タイムゾーン対応**

Cloud FunctionsはUTC環境で動作するため、すべての日付処理でJST（日本標準時）変換を厳密に実装した。`new Date()`をそのまま使うと9時間ずれるので、`Date.UTC()`を使ってUTCベースで日付を管理し、表示時にJSTに変換している。地味だけど、これを間違えると「昨日の支出が今日に記録される」みたいなバグが起きる。

**Googleカレンダー双方向同期**

毎日深夜3:00に同期ジョブが走り、カレンダー側で手動編集・削除した内容もFirestoreに自動反映される。カレンダーのイベントタイトルをパースして、カテゴリー・ユーザー名・金額を抽出する仕組みを作った。

タイトル形式は柔軟に設計していて、`[外食費用] 田中 ¥1,280`だけでなく、`外食 田中 1280円`のような省略形にも対応。`[]`の有無、`¥`や`円`の有無、カンマ区切りの有無など、ユーザーがカレンダー上で手動編集しても正しく認識できるようにしている。

**LINE上の対話形式UI**

テキストベースのLINEで擬似的な対話型インターフェースを実現するため、Firestoreでセッション管理を行っている。対話セッションは10分でタイムアウトし、途中で別のコマンドを送信すると自動的にキャンセルされる。1ユーザーにつき1つのセッションのみ保持する設計で、状態管理をシンプルに保っている。
