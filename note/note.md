# 月50円で同棲の"お金どうする問題"が消えた話

## 1. 同棲カップルの9割がぶつかる壁 —「お金どうする問題」

**「同棲のお金管理、あなたはどうしていますか？」**

「あれ、この前のスーパーの買い物って、どっちが払ったっけ？」

相棒と同棲を始めて数ヶ月が経ち、こんな会話が増えてきて、揉めることが度々あった。

同棲カップルにとって、**「お金の管理をどうするか」は避けて通れない最大級の問題**だと思う。好きで一緒に暮らし始めたのに、お金のことがきっかけで関係がギクシャクする——そんな話を自分たちだけでなく、周囲の同棲カップルからもよく聞いた。

「今月いくら使った？」「この前の外食、どっちが払ったっけ？」「月末の精算、計算が面倒だな……」

最初はざっくりでいいかと思っていた。しかし確認作業が積み重なると、それ自体がストレスになる。「あれ、この出費どっちだっけ？」や「返金されてなくない？」というやり取りが増えるたびに、喧嘩や気まずい空気が流れたことがあった。

最初はタイムツリー（TimeTree）に日々の支出を入力する方式をとっていた。最初の1~2週間は二人ともきちんと入力していた。しかし次の週くらいから入力が減り始めた。お互いに忙しくなる瞬間もあり、「あとでまとめて入れよう」になった。月末に精算しようとしたら、半分以上の支出が抜けていた時もあった。

ここで気づいたのは、**二人とも怠けていたわけではなく、「管理する仕組み自体がストレスになっていた」**ということだった。

- 入力が面倒で3日で開かなくなる
- エンジニアではない職業の相棒にとって、Googleスプレッドシート等の表計算ソフトをスマホから入力するのはめんどくさい
- 精算の計算を結局手動でやることになる

**仕組みがストレスを生んでいる**——これが根本的な問題だった。

---

## 2. エンジニアが"自分ごと"として向き合った理由

普段はWebエンジニアとして働いている。技術で問題を解決するのが仕事であり、趣味でもある。

だからこそ、相棒との間でお金のことをもっとスマートに管理したいと強く思った。

既存のアプリがうまくいかなかった原因を振り返った結果、一番の問題は**「相棒にとってのハードル」**だと気づいた。

自分はエンジニアなので巷にあるサービスであれば、ある程度使いこなせる。しかし相棒にとって、新しいサービスに足を踏み入れて、使い方を習得し、毎回入力するのはかなりのストレスがかかると言われた。

この言葉を聞いた時、**技術的な正解は対象者のニーズの上に成り立つ**のだと実感した。いくら高機能なアプリでも、相棒が使い続けられなければ意味がない。

**「相棒にとってのハードルをゼロにしたい」**

これが最大のモチベーションになった。

相棒は毎日LINEを使っている。朝起きたらLINEを開いて、仕事の連絡もLINEで連絡して、寝る前もLINEを見ている。

であれば、**LINEの中で完結すればいいのでは？**と考えた。

---

## 3. なぜLINE Botだったのか — 既に使っているツールの中で完結させる

最初からLINEに決めていたわけではない。SlackのBot、Webアプリの自作も候補に挙がった。

しかし、いずれも同じ壁にぶつかった。**相棒が普段使っていないサービスだった**ということ。Slackは仕事で使う人は多いが、相棒の日常のツールではない。Webアプリを自作するにしても、新しいURLをブックマークして、毎回ブラウザで開いて——という時点で、操作の習得コストが発生する。

タイムツリーの失敗から学んだのは、**「新しいツールの導入」自体がハードルになる**ということ。どんなに良いものを作っても、相棒が日常的に開かないプラットフォームでは続かない。

そこで残ったのがLINEだった。理由はシンプル。

**相棒が既に日常的に使っているサービスだから。**

これに尽きる。

わざわざ別のプラットフォームを介さず、普段の延長線上で使える。「新しいアプリ入れて」と頼む必要がない。これなら継続できると確信した。

導入方法も非常にシンプルで、グループトークにBotを追加するだけ。LINEのグループを作って、そこにBotを招待する。それだけで二人で家計管理ができるようになる。

通知もLINEで届くので見逃さない。月末の収支の集計レポートも、毎朝の予定通知も、全部LINEに届く。別のアプリを開く必要は一切ない。

開発を進める中で強く感じたのは、**「新しいアプリを作る」よりも「既存の習慣に乗せる」方がはるかに定着率が高い**ということだった。技術的にはネイティブアプリの方が自由度は高いが、使ってもらえなければ意味がない。

## 4. 二人で決めた精算ルール — Botが自動化したもの

ここで紹介するルールは、Botのために新しく作ったものではない。同棲を始めた頃に二人で「お金の管理どうする？」と話し合って決めたもので、Bot開発よりずっと前から運用していた。

このルールがすでにあったからこそ、そのままBotに組み込むことができた。つまり、**先にルールがあって、後からBotがそれを自動化した**という順番になる。

振り返ると、この「先にルールを決める」というステップが一番大事だったと感じている。どんなに便利なツールでも、二人の間で「こうしよう」という合意がなければ、ツールは使われなくなる。

> **エンジニアでなくても、このルールだけで効果がある。** LINE Botを使わなくても、半月担当制・予算制・精算期限——この3つのルールを二人で決めるだけで、お金の話のストレスはかなり減ると思う。

### 半月担当制 —「今日どっちが払う？」がゼロになる

これが最も効果の大きい仕組みだと感じている。

| 期間 | 担当 | 役割 |
|------|------|------|
| 月の前半（1日〜15日） | Aさん | 外食の支払い担当 |
| 月の後半（16日〜月末） | Bさん | 外食の支払い担当 |

担当期間中は、その人が外食費をすべて管理する。たったこれだけで、レジ前での「どっちが出す？」問題が完全になくなった。期間毎に担当するのでお互いがフェアであり、「この期間は自分の担当だから自分が払う」と迷いがゼロになる。

最初は「本当にこれだけでうまくいくのか？」と半信半疑だったが、実際に始めてみると、外食の会計時のストレスが嘘のようになくなった。

### 予算制 — 残高が見えるから自然と意識が変わる

外食費用は一人月X万円の予算制（`@予算`コマンドで金額は変更できる）。

使うたびに残高が減っていくので、ゲームのHP管理のような感覚になる。残高が少なくなると自然と「今月はセーブしよう」という意識が生まれ、月末に残高が多いと「今月は節約できた」とポジティブな話題になる。予算を超過した場合は超過分が担当者の自己負担になるので、**自分ごとで管理する仕組み**になっている。

運用して気づいたのは、**予算を「見える化」するだけで、二人のお金に対する意識が自然と変わった**ということ。制限されている感覚ではなく、ゲーム感覚で楽しめるようになった。

```
💰 外食残高

田中: ￥35,000
鈴木: ￥42,000

📅 今月の予算: ￥50,000/人
```

### 買い物・旅行 — 各自が払って月末にBotが自動精算

買い物や旅行の費用は、普段はそれぞれ気にせず支払えばよい。月末にBotが自動で差額を計算して、誰がいくら返せばいいかを教えてくれる。

自動で計算されるので、「今月どっちがいくら払ったか」を自分で計算する必要は一切ない。

```
【🛒 買い物費用 - 精算】
田中 合計: ￥25,000
鈴木 合計: ￥28,000
差額: ￥3,000

→ 田中が鈴木に ￥1,500 を返金してください🙏
```

### 精算の期限

> **精算は翌月15日まで。それを過ぎたら無効。**

期限を決めることで「来月やろう」「再来月でいいか」とズルズル先延ばしにならない。このルールがあるおかげで、お金のことで感情的になることがほぼなくなった。

このルールがあることで、残りの予算を鑑みた行動ができるようになった。「期限までに精算する」という意識があるからこそ、計画的にお金を使う習慣が自然と身についた。

### 月末の自動レポート

15日に中間レポート、月末に最終レポートが自動でLINEに届く。

最終レポートには、外食の貯金額（予算5万円 − 使った金額）と買い物・旅行の精算額がまとめて表示される。二人でレポートを見ながら「今月はこうだったね」と振り返るのがちょっとした楽しみになった。

```
━━━━━━━━━━━━━━━━━━━━
📅 月間サマリー
━━━━━━━━━━━━━━━━━━━━

【💰 外食費用 - 貯金額】
田中: ￥25,000 使用 → 貯金成功 ￥25,000 🎉
鈴木: ￥21,000 使用 → 貯金成功 ￥29,000 🎉

【🛒 買い物費用 - 精算】
田中 合計: ￥25,000
鈴木 合計: ￥28,000
差額: ￥3,000

→ 田中が鈴木に ￥1,500 を返金してください🙏
```

### まとめると

| ルール | 内容 |
|--------|------|
| 🍽️ 外食の支払い | 半月ごとに担当者を交代 |
| 💰 外食の予算 | 一人月5万円。超過分は担当者負担 |
| 🛒 買い物・旅行 | 各自が払って月末に差額を精算 |
| ⏰ 精算の期限 | 翌月15日まで。過ぎたら無効 |

> もし今、お金の管理方法に迷っているなら、まずはこの4つのルールだけ試してみてほしい。ツールは後からいくらでも変えられるが、「二人で決めたルール」は何よりも強い。

## 5.「家計ぼっと」の全体像 — LINEだけで家計管理が完結する仕組み

名前は**「家計ぼっと」**。LINEのグループトークに住んでいるBot。

できることを一言で言うと、**LINEで画像を送ったり対話形式で入力するだけで、収支を管理できる**。

### 4つの支出カテゴリー

自分が作成したBotでは、家計簿の勘定科目を4つのカテゴリーで管理している。

| カテゴリー | 内容 |
|-----------|------|
| 🍽️ 外食費用 | レストラン・飲食店での食事。一人ずつの予算制 |
| 🛒 買い物費用 | スーパー・コンビニでの買い物。月末精算 |
| ✈️ 旅行費用 | 交通費・宿泊費など旅行の支出。月末精算 |
| 🔄 サブスク | 家賃・Netflix・定期購買など固定費。月初に自動登録 |

最初は「外食」と「それ以外」の2カテゴリーだけだった。しかし運用していくうちに、旅行費用を分けたい、サブスクも管理したいというニーズが出てきて、現在の4カテゴリーに落ち着いた。**実際に使いながら育てていった**形になる。

### できること一覧

| 機能 | 内容 |
|------|------|
| 📸 レシート画像で自動登録 | レシートやPayPay・Uber Eats等のスクショを送るだけでAIが解析・登録 |
| ✏️ テキストで手入力登録 | レシートがないときは対話形式やワンライナーで手動登録 |
| 🔍 履歴・集計の確認 | 直近の支出一覧や、月別のカテゴリー別集計を確認 |
| 🗑️ 削除・修正 | 間違えて登録した支出を対話形式で取り消し |
| 💰 外食残高の管理 | 予算制で「あといくら使えるか」をリアルタイムに把握 |
| 📊 中間・月末レポート | 15日に中間レポート、月末に精算額・貯金額をまとめた最終レポートを自動送信 |
| 📅 予定リマインド | Googleカレンダーに予定を登録し、当日の朝7:00にLINEで通知 |
| 🔄 サブスク自動登録 | Netflix・家賃・定期購買などを月初に自動で記録 |
| 📆 Googleカレンダー連携 | 支出がカレンダー上に色分けで表示。カレンダー側の編集もBotに自動反映 |

メインの使い方はレシート画像を送っての自動登録だが、レシートがないときは手入力でも登録できる。中間・月末のレポートは自動で届くし、予定リマインドやサブスクの自動登録まで対応している。**家計管理に必要なことが、すべてLINEの中で完結する。**

## 6. レシートを撮って送るだけ — AIが自動で解析してくれる

一番よく使う機能は、レシートやPayPayの画面をスクショしてLINEに送信すること。操作はこれだけで完了する。

送信すると、Gemini API（GoogleのAI）が自動で以下の情報を解析してくれる。

- **日付** — レシートに書かれている日付
- **金額** — 支払った金額
- **カテゴリー** — 外食/買い物/旅行を店舗名から自動判定
- **お店の名前** — どこで使ったか

登録が終わると、Botが即座に返答してくれる。

```
✅ 登録しました！
📝 外食費用: ￥1,280（田中）
🏪 サイゼリヤ
📅 2024/12/03

💰 田中の外食残高: ￥48,720
```

外食費用の場合は残高も一緒に返ってくるので、「今月あといくら使えるか」がすぐわかる。

Googleカレンダーにも自動で登録される。事前に色分けの定義がされていて、外食費用はフラミンゴ（ピンク系）、買い物費用はバナナ（黄色系）、旅行費用はラベンダー（紫系）で表示される。

対応しているのはレシートだけではなく、**PayPay・LINE Pay・Uber Eats・クレカの明細画面のスクショ**にも対応している。電子決済アプリの支払い完了画面を撮って送れば、同じように自動解析してくれる。

この画像解析の精度を上げるのに一番苦労した。最初はレシートの日付を読み間違えたり、テイクアウト専門店を「外食」に分類してしまったりすることがあった。プロンプトに店舗ごとのカテゴリー判定ルールを細かく書き込むことで、実用レベルの精度まで改善できた。

## 7. 他にもこんな使い方ができる

### テキストで手動登録（@追加・@旅行）

レシートがないときは、対話形式で1つずつ入力できる。Botが「カテゴリーは？」「金額は？」と順番に聞いてくるので、答えていくだけでよい。

急いでいるときは、ワンライナーでも登録できる。

```
@追加 @自分 外食費用 1280
```

`@自分`と入力すると、送信者の名前が自動で入る。フルネームを打つ必要はない。LINEのメンション機能（@ユーザー名）で相棒を指定することもできる。

旅行費用は`@旅行`コマンドで、画像解析と手動入力のどちらかを選べる。

### 支出の確認（@履歴・@集計）

`@履歴`で直近の支出を一覧表示。`@集計`（`@レポート`でも同じ）で今月のカテゴリー別集計を確認できる。

```
📋 直近の支出履歴

1. 12/03 [外食費用] 田中 ￥1,280
2. 12/02 [買い物費用] 鈴木 ￥2,500
3. 12/01 [外食費用] 田中 ￥980
...
```

過去の月も`@集計 2024/12`のように年月を指定すれば確認できる。

### 予定管理（@予定）

```
@予定 ディナー 12/25 19:00
```

家計簿だけでなく、二人のスケジュール管理にも使える。Googleカレンダーに予定が登録されて、当日の朝7:00にLINEで通知が届く。カンマ区切りで複数人の予定も登録できる。

### サブスク・家賃の自動登録

Netflix、Spotifyなど毎月固定の支払いや、週単位の定期便（Amazon定期おトク便など）を`@サブスク追加`で登録しておくと、月初めに自動で「買い物費用」として記録してくれる。

家賃も`@家賃追加`で一度登録すれば、毎月の月末にGoogleカレンダーへ自動登録される。

```
🔄 サブスク自動登録完了
━━━━━━━━━━━━━━━

2024年12月分のサブスクを登録しました

✅ Netflix（鈴木・¥1,490・12/15）
✅ Amazon定期便 猫砂（田中・¥1,980・12/10）
```

### 毎朝の予定通知

毎朝7:00にその日の予定をLINEで通知してくれる。朝起きてLINEを開くと、今日の予定が見える。

```
📅 本日の予定 (12/15)

1. 👤 田中
   📝 会議

2. 👤 鈴木
   📝 買い物
```

### 削除・修正も対話形式で簡単

間違えて登録しても`@削除`で対話形式で取り消せる。ワンライナー（`@削除 @自分 外食費用 1280`）でも対応している。

```
🗑️ 以下の支出を削除しました

📅 12/03
📝 外食費用: ￥1,280（田中）
🏪 サイゼリヤ

💰 田中の外食残高: ￥36,280（+￥1,280）
```

外食費用を削除した場合は、残高も正しく復元される。

## 8. Googleカレンダー連携 — 支出が「見える」ようになる

個人的にお気に入りの機能が、Googleカレンダーとの連携。

支出がカレンダー上に色分けで並ぶので、「今月いつ何に使ったか」が一目瞭然。

| カテゴリー | カレンダーの色 |
|-----------|-------------|
| 🍽️ 外食費用 | フラミンゴ（ピンク系） |
| 🛒 買い物費用 | バナナ（黄色系） |
| ✈️ 旅行費用 | ラベンダー（紫系） |
| 🔄 サブスク | バジル（緑） |

さらに、カレンダー上で直接イベントを編集・削除することもできる。毎日深夜3:00にGoogleカレンダーとBotのデータベース間で同期が走るので、LINEから登録した支出はカレンダーに反映され、カレンダー側で手動編集・削除した内容もBotのデータに自動で反映される。

月末にカレンダーをざっと見返すだけで、お金の使い方のパターンが理解できるようになる。「先月は旅行多かったな」「今月は外食控えめだった」——数字だけのレポートでは得られない、視覚的な気づきがある。

この機能を入れて良かったと感じるのは、**「振り返り」のハードルが下がった**こと。わざわざアプリを開いて集計を見なくても、カレンダーをスクロールするだけで1ヶ月のお金の使い方が把握できる。

## 9. 使ってみてどうだったか — Before/Afterと運用の変化

### 導入の瞬間

相棒に「LINEグループにBot入れたから、レシート送ってみて」と伝えるだけで始められた。

新しいアプリのインストールもアカウント登録も一切なし。相棒もすぐに使い始めてくれた。このハードルの低さが一番大きかったと思う。

### Before（Bot導入前）

- 月末にレシートの山を二人で整理していた
- 「この外食、どっちが払った？」で何度も気まずくなった
- 精算を先延ばしにして、2ヶ月分まとめてやったことがある
- 家計簿アプリを入れても1週間で開かなくなった

### After（Bot導入後）

- レシートを送るだけ。月末の精算作業はゼロになった
- 残高が見えるから「今月はあと○円だね」と自然な会話が生まれた
- **お金の話が「しなきゃいけないこと」から「振り返りの楽しみ」に変わった**

### 運用して数ヶ月で変わったこと

導入直後と数ヶ月後では、使い方にも変化があった。

最初は「記録するためのツール」だったが、運用を重ねるうちに**「二人のお金に対する意識を揃えるツール」**に変わっていった。月末レポートを一緒に見て「今月は外食多かったね」「来月はもう少しセーブしよう」と自然に話し合えるようになった。

また、Botの改善も継続的に行っている。相棒からの「ここが使いにくい」「こういう機能がほしい」というフィードバックを受けて、サブスク自動登録機能や予定通知機能を後から追加した。**使う人の声を聞きながら育てていくプロセス自体が楽しい**と感じている。

## 10. おわりに —「ないなら、作ればいい」し、作れなくてもルールがある

大きなサービスを作ることだけが技術ではない。身近な人の「面倒だ」を1つ解決する。それだけで、生活の質は確実に上がる。

もし今、同棲やシェアハウスでお金の管理に困っているなら、2つの選択肢がある。

**1. まずは「二人に合ったルール」を決める（全員向け）**

LINE Botを使わなくても、ここで紹介した3つのルール——半月担当制、予算制、精算期限——だけでも試してみてほしい。ルールがあるだけで、お金の話のストレスはかなり減る。大事なのは、**完璧なアプリを探し続けるより、「二人で合意したルール」を持つこと**。

**2. 自動化に挑戦してみる（エンジニア向け）**

もし技術に興味があるなら、ソースコードを公開しているのでぜひ覗いてみてほしい。フォークして自分たちのルールに合わせてカスタマイズするのも面白いと思う。

### 11.3 なぜGCPを選んだか

- **サーバーレスで管理不要** — サーバーの面倒を見なくていい
- **無料枠が充実** — Cloud Functions + Firestore + Cloud Scheduler がほぼ無料枠内で収まる
- **Google Calendar APIとの親和性** — 同じGoogleエコシステム内でスムーズに連携できる

### 11.4 なぜFirestoreを選んだか（Cloud SQLとの比較）

データベースの選定で一番悩んだのがここ。Cloud SQL（MySQL/PostgreSQL）も候補に挙がったが、最終的にFirestoreを選んだ。

| 観点 | Firestore | Cloud SQL |
|------|-----------|-----------|
| 月額コスト | $0（無料枠内） | $15〜20（最小構成でも） |
| インスタンス管理 | 不要（完全サーバーレス） | 必要（常時起動） |
| Cloud Functions連携 | ネイティブ対応 | VPC Connector必要（追加コスト） |
| スケーリング | 自動 | 手動設定が必要 |

**年間のコスト差は歴然で、Firestoreなら$0、Cloud SQLは$180〜240（約¥27,000〜36,000）。** 個人プロジェクトにはFirestoreが圧倒的にコスト優位だった。

本システムのデータ構造は「各支出が独立したドキュメント」で、複雑なJOINも不要。Firestoreのドキュメント型データベースと非常に相性が良かった。

### 11.7 技術的に工夫したポイント

**Gemini APIのプロンプト設計**

レシートだけでなく、PayPay・Uber Eats・クレカ明細など多様な支払い画面に対応するため、プロンプトに詳細なカテゴリー判定ルールを記載した。

例えば、「ファミリーマート→買い物費用」「サイゼリヤ→外食費用」「JR東日本→旅行費用」のように、店舗名からカテゴリーを自動判定するルールを組み込んでいる。テイクアウト専門店（丼丸、ほっともっとなど）は外食ではなく買い物費用に分類するなど、実際の使い方に沿った細かいルール設計をしている。

**タイムゾーン対応**

Cloud FunctionsはUTC環境で動作するため、すべての日付処理でJST（日本標準時）変換を厳密に実装した。`new Date()`をそのまま使うと9時間ずれるので、`Date.UTC()`を使ってUTCベースで日付を管理し、表示時にJSTに変換している。目立たない部分ではあるが、これを間違えると「昨日の支出が今日に記録される」というバグが発生する。

**Googleカレンダー同期**

毎日深夜3:00に同期ジョブが走り、カレンダー側で手動編集・削除した内容もFirestoreに自動反映される。カレンダーのイベントタイトルをパースして、カテゴリー・ユーザー名・金額を抽出する仕組みを作った。

タイトル形式は柔軟に設計していて、`[外食費用] 田中 ¥1,280`だけでなく、`外食 田中 1280円`のような省略形にも対応。`[]`の有無、`¥`や`円`の有無、カンマ区切りの有無など、ユーザーがカレンダー上で手動編集しても正しく認識できるようにしている。

**LINE上の対話形式UI**

テキストベースのLINEで擬似的な対話型インターフェースを実現するため、Firestoreでセッション管理を行っている。対話セッションは10分でタイムアウトし、途中で別のコマンドを送信すると自動的にキャンセルされる。1ユーザーにつき1つのセッションのみ保持する設計で、状態管理をシンプルに保っている。
