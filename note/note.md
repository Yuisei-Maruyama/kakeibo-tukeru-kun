# 月50円で同棲の"お金どうする問題"が消えた話

## 1. 一緒に暮らし始めたら必ずぶつかる壁 —「お金どうする問題」

**「誰かと一緒に暮らす中での金銭の管理、あなたはどうしていますか？」**

「あれ、この前のスーパーの買い物って、どっちが払ったっけ？」

相棒と同棲を始めて1年が経ち、こんな会話が増えてきて、揉めることが度々あった。

誰かと一緒に暮らすということは、生活費を共有するということでもある。「お金の管理をどうするか」は、共同生活において避けて通れない問題だと思う。せっかく一緒に暮らし始めたのに、お金のことがきっかけで関係がギクシャクしてしまう。そんな話を自分たちだけでなく、周囲からもよく聞いていた。

「今月いくら使った？」「この前の外食、どっちが払ったっけ？」「月末の精算、計算が面倒だな……」

最初は食事代を毎回交代で支払う方式をとっていた。しかし店によって金額はバラバラなので、どちらか一方の負担額が大きくなることがあった。「前回は自分が高い店だったのに」。そんな小さな不安が募ることがあった。

次にTimeTreeに日々の支出を入力する方式を試した。最初の1~2週間は二人ともきちんと入力していた。しかし次の週くらいから入力が減り始めた。お互いに忙しくなる瞬間もあり、「あとでまとめて入れよう」になった。結局、決めていた期限内に入力できず、相棒の気を悪くさせてしまったこともあった。

二人とも怠けていたわけではないのに、なぜ続かないのかを振り返った。すると、交代制もTimeTreeも同じパターンで失敗していたことに気づいた。

- 交代制は金額差を気にするストレスが生まれた
- TimeTreeは入力の手間が壁になり、数週間で続かなくなった
- どちらの方法でも、最終的な精算は手計算に頼ることになり、計算ミスや含め忘れのリスクが常に伴った

方法を変えても結局うまくいかなかった。この一連の失敗を俯瞰してみると、**問題の根本は「やり方」ではなく、金銭管理の「仕組み」そのものにあった**ことが見えてきた。

---

## 2. 仕組みを作り変える

仕組みそのものに問題があるなら、自分で仕組みを作ってしまえばいい。
自分は本業でWebエンジニアとして働いていて、技術で問題を解決するのが仕事であり、趣味でもある。
目の前の問題を技術で解決できる手段が、自分にはあった。

身近な課題を前にして、「これは技術で解けるのでは？」という発想がいくつか浮かんだ。

ただ、何を作り直すべきかを考えるために、もう一歩踏み込んで「なぜ続かなかったのか」を振り返った。そこで見えてきたのは、一番の問題として **「相棒にとっての心理的ハードルがあった」**ということだった。

自分はエンジニアなので、触れたことのないサービスでも好奇心を持って使うことができる。しかし相棒にとっては、新しいサービスの使い方を覚えて毎回入力すること自体がストレスだと言われた。

この言葉を聞いた時、**技術的な正解は対象者のニーズの上に成り立つ**のだと改めて実感した。いくら高機能なアプリでも、ユーザーが継続して使い続けられなければ意味がない。

**「相棒にとっての心理的ハードルをゼロにしたい」**

これが最大のモチベーションになった。

相棒は毎日LINEを使っている。朝起きたらLINEを開いて、仕事の連絡もLINEで連絡して、寝る前もLINEを見ている。

であれば、LINEの中で完結すればいいのでは？と考えた。

## 3. 既に使っているツールの中で完結させる

最終的にLINEで完結させるという結論に至ったが、実際にはここに辿り着くまでに他の選択肢も検討していた。SlackのBot、Webアプリの自作も候補に挙がった。

しかし、いずれも同じ壁にぶつかった。相棒が普段使っていないサービスだったということ。Slackは仕事で使う人は多いが、相棒の日常のツールではない。Webアプリを自作するにしても、新しいURLをブックマークして、毎回ブラウザで開くという時点で、操作の習得コストが発生する。

これまでの失敗から学んだのは、**「新しいツールの導入」自体が心理的な壁になる**ということ。どんなに良いものを作っても、相棒にとって親近感のないプラットフォームでは続かない。

そこで残ったのがLINEだった。理由はシンプル。

**相棒が既に日常的に使っているサービスだから。**

これに尽きる。

わざわざ別のプラットフォームを介さず、普段の延長線上で使える。「新しいアプリ入れて」と頼む必要がない。これなら継続できると確信した。

導入もLINEグループにBotを招待するだけ。通知も月末の収支レポートも予定リマインドも、すべてLINEに届く。別のアプリを開く必要は一切ない。

開発を進める中で強く感じたのは、「新しいアプリを作る」よりも「既存の習慣に乗せる」方がはるかに定着率が高いということだった。技術的にはネイティブアプリの方が自由度は高いが、使ってもらえなければ意味がない。

## 4. 二人で決めた精算ルール — Botが自動化したもの

ここで紹介するルールは、Botのために新しく作ったものではない。同棲を始めた頃に二人で「お金の管理どうする？」と話し合って決めたもので、Bot開発よりずっと前から運用していた。

このルールがすでにあったからこそ、そのままBotに組み込むことができた。つまり、**先にルールがあって、後からBotにそれを自動化させた**という順番になる。

振り返ると、この「先にルールを決める」というステップが一番大事だったと感じている。これは仕事でも同じで、目標を決めずに闇雲に進めても、何が正解か分からず曖昧なものになる。目標の解像度が高ければ、正しいプロセスでやるべきことを遂行できる。同棲の金銭管理もまったく同じで、二人の間で「このルールで取り決めよう」という要件定義がしっかりできていれば、ツールは日常的に使われるものになる。

> **エンジニアでなくても、このルールだけで効果がある。** LINE Botを使わなくても、半月担当制・予算制・精算期限、このように予めルールを決めるだけで、お金の話のストレスはかなり減ると思う。

### 半月担当制 —「今日どっちが払う？」がゼロになる

これが最も効果の大きい仕組みだと感じている。

![半月担当制テーブル](note/img/table/01_table.png)

担当期間中は、その人が外食費をすべて管理する。たったこれだけで、レジ前での「どっちが出す？」問題が完全になくなった。期間毎に担当するのでお互いが必ずフェアな状態であり、「この期間は自分の担当だから自分が払う」と迷いがゼロになる。

最初は「本当にこれだけでうまくいくのか？」と半信半疑だったが、実際に始めてみると、外食の会計時のストレスが嘘のようになくなった。

### 予算制 — 残高が見えるから自然と意識が変わる

外食費用は一人月X万円の予算制。金額は二人で話し合って決めた。

使うたびに残高が減っていくので、ゲームのHP管理のような感覚になる。残高が少なくなると自然と「今月はセーブしよう」という意識が生まれ、月末に残高が多いと「今月は節約できた」とポジティブな話題になる。予算を超過した場合は超過分が担当者の自己負担になるので、自分ごとで管理する仕組みになっている。

運用して気づいたのは、**予算を「見える化」するだけで、二人のお金に対する意識が自然と変わった**ということ。制限されている感覚ではなく、ゲーム感覚で楽しめるようになった。

### 買い物・旅行 — 各自が払って月末に精算

買い物や旅行は各自が気にせず支払い、月末にそれぞれの合計額を出して差額を精算する。「今回どっちが払う？」を毎回気にする必要がなくなった。

### 精算の期限

> **精算は翌月15日まで。それを過ぎたら無効。**

期限を決めることで「来月やろう」「再来月でいいか」とズルズル先延ばしにならない。このルールがあるおかげで、お金のことで感情的になることがほぼなくなった。

このルールがあることで、残りの予算を鑑みた行動ができるようになった。「期限までに精算する」という意識があるからこそ、計画的にお金を使う習慣が自然と身についた。

### まとめると

| ルール | 内容 |
|--------|------|
| 🍽️ 外食の支払い | 半月ごとに担当者を交代 |
| 💰 外食の予算 | 一人月X万円。超過分は担当者負担 |
| 🛒 買い物・旅行 | 各自が払って月末に差額を精算 |
| ⏰ 精算の期限 | 翌月15日まで。過ぎたら無効 |

## 5.「家計ぼっと」の全体像 — LINEだけで家計管理が完結する仕組み

**「家計ぼっと」** はLINEのグループトークで動く家計管理Bot。LINEグループに友達追加するだけで使い始められ、画像を送るか対話形式で入力するだけで収支を管理できる。

### 4つの支出カテゴリー

支出は4つのカテゴリーで管理している。

| カテゴリー | 内容 |
|-----------|------|
| 🍽️ 外食費用 | レストラン・飲食店での食事。一人ずつの予算制 |
| 🛒 買い物費用 | スーパー・コンビニでの買い物。月末精算 |
| ✈️ 旅行費用 | 交通費・宿泊費など旅行の支出。月末精算 |
| 🔄 サブスク | 家賃・Netflix・定期購買など固定費。月初に自動登録 |

<!-- ここに note/img/table/03_table.png を入れる -->

最初は「外食費用」と「買い物費用」の2カテゴリーだけだった。しかし運用していくうちに、旅行費用を分けたい、サブスクも管理したいというニーズが出てきて、現在の4カテゴリーに落ち着いた。実際に使いながら育てていった形になる。

### 集計レポート — 月の振り返りが自動で届く

毎月15日と月末に、その期間の集計レポートがLINEに自動で届く。自分で集計する必要は一切ない。

**中間レポート（毎月15日）** では、前半の外食費用・買い物費用・旅行費用をカテゴリー別に表示。外食の残高もわかるので、後半の使い方を調整できる。

**月末レポート** では、さらに踏み込んだ内容が届く。

- **外食費用の貯金額** — 予算から実際の使用額を引いた残りが「貯金」として表示される。「今月は二人で○円節約できた」と自然にポジティブな話題になる
- **買い物・旅行の精算額** — それぞれの合計額の差額を自動で計算し、「○○さんが○○さんに△円払う」と精算内容を具体的に提示してくれる。手計算のミスや含め忘れがゼロになった

```
📊 月末レポート（12/1〜12/31）

🍽️ 外食費用
  田中: ¥38,200（残高: ¥11,800）
  鈴木: ¥42,500（残高: ¥7,500）

💰 外食の貯金額
  田中: ¥11,800
  鈴木: ¥7,500

🛒 買い物費用
  田中: ¥15,200
  鈴木: ¥22,800
  → 田中が鈴木に ¥3,800 精算

✈️ 旅行費用
  田中: ¥8,000
  鈴木: ¥12,000
  → 田中が鈴木に ¥2,000 精算
```

このレポートがあることで、月末の「レシートの山を二人で整理する時間」が完全になくなった。レポートを見ながら「今月は外食多かったね」「来月はもう少し節約しよう」と自然に振り返りの会話が生まれるようになった。

メインはレシート画像の自動登録。手入力、集計確認、自動レポート、予定リマインド、サブスク・家賃自動登録、カレンダー双方向同期まで、家計管理のすべてがLINEの中で完結する。

## 6. レシートを撮って送るだけ — AIが自動で解析してくれる

レシートやPayPayの画面をスクショしてLINEに送る。これだけで完了する。

Gemini API（GoogleのAI）が自動で以下を解析してくれる。

- **日付** — レシートに書かれている日付
- **金額** — 支払った金額
- **カテゴリー** — 外食/買い物/旅行を店舗名から自動判定
- **お店の名前** — どこで使ったか

登録が完了すると即座に返答が届く。

```
✅ 登録しました！
📝 外食費用: ￥1,280（田中）
🏪 サイゼリヤ
📅 2024/12/03

💰 田中の外食残高: ￥48,720
```

外食費用なら残高も一緒に返ってくるので、「あといくら使えるか」がすぐわかる。Googleカレンダーにも自動登録され、外食はフラミンゴ（ピンク系）、買い物はバナナ（黄色系）、旅行はラベンダー（紫系）で色分け表示される。

レシートだけでなく、**PayPay・LINE Pay・Uber Eats・クレカ等の明細画面のスクリーンショット**にも対応している。

この画像解析の精度を上げるのに一番苦労した。最初はレシートの日付を読み間違えたり、テイクアウト専門店を「外食」に分類してしまったりすることがあった。Geminiのモデルを上げることやプロンプトに店舗ごとのカテゴリー判定ルールを細かく書き込むことで、実用レベルの精度まで改善できた。

## 7. 他にもこんな使い方ができる

### テキストで手動登録（@追加・@旅行）

レシートがないときは対話形式で入力できる。Botが「カテゴリーは？」「金額は？」と順番に聞いてくる。急いでいるときはワンライナーでも登録できる。

```
@追加 @自分 外食費用 1280
```

`@自分`で送信者の名前が自動入力される。LINEのメンション機能で相棒を指定することもできる。旅行費用は`@旅行`コマンドで、画像解析と手動入力を選べる。

### 支出の確認（@履歴・@集計）

`@履歴`で直近の支出一覧、`@集計`（`@レポート`でも可）で今月のカテゴリー別集計を確認できる。

```
📋 直近の支出履歴

1. 12/03 [外食費用] 田中 ￥1,280
2. 12/02 [買い物費用] 鈴木 ￥2,500
3. 12/01 [外食費用] 田中 ￥980
...
```

過去の月も`@集計 2024/12`のように年月を指定すれば確認できる。

### 予定管理（@予定）

```
@予定 ディナー 12/25 19:00
```

二人のスケジュール管理にも使える。Googleカレンダーに登録され、当日の朝7:00にLINEで通知が届く。

### サブスク・家賃の自動登録

`@サブスク追加`で毎月固定の支払い（Netflix、Spotifyなど）や定期便を登録しておくと、月初めに自動で記録される。家賃も`@家賃追加`で一度登録すれば毎月自動で登録される。

```
🔄 サブスク自動登録完了
━━━━━━━━━━━━━━━

2024年12月分のサブスクを登録しました

✅ Netflix（鈴木・¥1,490・12/15）
✅ Amazon定期便 猫砂（田中・¥1,980・12/10）
```

### 毎朝の予定通知

毎朝7:00に、その日の予定がLINEに届く。

```
📅 本日の予定 (12/15)

1. 👤 田中
   📝 会議

2. 👤 鈴木
   📝 買い物
```

### 削除・修正も対話形式で簡単

間違えて登録しても`@削除`で取り消せる。

```
🗑️ 以下の支出を削除しました

📅 12/03
📝 外食費用: ￥1,280（田中）
🏪 サイゼリヤ

💰 田中の外食残高: ￥36,280（+￥1,280）
```

外食費用を削除した場合は、残高も正しく復元される。

### コマンドチートシート — 「どんなコマンドだっけ？」をゼロにする

## 9. 使ってみてどうだったか — Before/Afterと運用の変化

### 導入の瞬間

「LINEグループにBot入れたから、レシート送ってみて」。これだけで始められた。アプリのインストールもアカウント登録も不要。この導入の手軽さが一番大きかった。

相棒や周囲の反応も良かった。LINEという多くの人が慣れ親しんだツールで完結する点に可能性を感じてもらえたし、自動化で金銭管理のストレスから解放される部分にもポジティブな意見をもらった。

[画像: 相棒がレシートを送信して、Botが即座に返答している実際のやり取り]

### Before（Bot導入前）

- 月末にレシートの山を二人で整理していた
- 「この外食、どっちが払った？」で何度も気まずくなった
- 精算を先延ばしにして、2ヶ月分まとめてやったことがある
- 家計簿アプリを入れても1週間で開かなくなった

### After（Bot導入後）

- レシートを送るだけ。月末の精算作業はゼロになった
- 残高が見えるから「今月はあと○円だね」と自然な会話が生まれた
- **お金の話が「しなきゃいけないこと」から「自動的に精算できること」に変わった**

### 日常に溶け込んだ瞬間

今では外食後にお互いが無意識にレシートを撮ってBotに送っている。「あといくら使える？」と残高を確認するのも、二人の日常会話の一部になった。意識して「家計簿をつけよう」と思う瞬間がなくなった。それがこのツールが日常に浸透した証拠だと思う。

### 運用して数ヶ月で変わったこと

最初は「記録ツール」だったが、数ヶ月後には**「二人のお金に対する意識を揃えるツール」**に変わっていた。月末レポートを一緒に見て「今月は外食多かったね」「来月はセーブしよう」と自然に話し合えるようになった。

相棒からの「ここが使いにくい」「こういう機能がほしい」というフィードバックで、サブスク自動登録や予定通知を後から追加した。どんなプロダクトでもそうだと思うが、ユーザーとの距離感が近いプロダクトは、使う人の声を聞きながら育てていくプロセス自体に良さがあると感じている。

### 11.7 技術的に工夫したポイント

**Gemini APIのプロンプト設計**

レシートだけでなく、PayPay・Uber Eats・クレカ明細など多様な支払い画面に対応するため、プロンプトに詳細なカテゴリー判定ルールを記載した。

例えば、「ファミリーマート→買い物費用」「サイゼリヤ→外食費用」「JR東日本→旅行費用」のように、店舗名からカテゴリーを自動判定するルールを組み込んでいる。テイクアウト専門店（丼丸、ほっともっとなど）は外食ではなく買い物費用に分類するなど、実際の使い方に沿った細かいルール設計をしている。

**タイムゾーン対応**

Cloud FunctionsはUTC環境で動作するため、すべての日付処理でJST（日本標準時）変換を厳密に実装した。`new Date()`をそのまま使うと9時間ずれるので、`Date.UTC()`を使ってUTCベースで日付を管理し、表示時にJSTに変換している。目立たない部分ではあるが、これを間違えると「昨日の支出が今日に記録される」というバグが発生する。

**Googleカレンダー同期**

毎日深夜3:00に同期ジョブが走り、カレンダー側で手動編集・削除した内容もFirestoreに自動反映される。カレンダーのイベントタイトルをパースして、カテゴリー・ユーザー名・金額を抽出する仕組みを作った。

タイトル形式は柔軟に設計していて、`[外食費用] 田中 ¥1,280`だけでなく、`外食 田中 1280円`のような省略形にも対応。`[]`の有無、`¥`や`円`の有無、カンマ区切りの有無など、ユーザーがカレンダー上で手動編集しても正しく認識できるようにしている。

**LINE上の対話形式UI**

テキストベースのLINEで擬似的な対話型インターフェースを実現するため、Firestoreでセッション管理を行っている。対話セッションは10分でタイムアウトし、途中で別のコマンドを送信すると自動的にキャンセルされる。1ユーザーにつき1つのセッションのみ保持する設計で、状態管理をシンプルに保っている。
