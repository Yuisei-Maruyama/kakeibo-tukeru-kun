## 2. 処理フロー

### 2.1 画像送信時の処理フロー

```
┌──────┐    ┌──────────┐    ┌──────────────┐    ┌─────────┐    ┌──────────┐
│ LINE │───▶│ Webhook  │───▶│ Gemini API   │───▶│Firestore│───▶│ Calendar │
│      │    │ Handler  │    │ (画像解析)   │    │ (保存)  │    │ (登録)   │
└──────┘    └──────────┘    └──────────────┘    └─────────┘    └──────────┘
    ▲                                                              │
    │                                                              │
    └──────────────────── LINE Reply ──────────────────────────────┘
```

**詳細フロー:**

1. ユーザーが LINE で画像を送信
2. LINE Webhook が Cloud Functions を呼び出し
3. 画像を取得し Gemini API で解析
4. 解析結果（日付・カテゴリー・金額・店舗名）を抽出
5. Firestore に支出データを保存
6. Google Calendar にイベントを登録
7. 残高を計算して LINE に返答

### 2.2 定期集計通知の処理フロー

```
┌───────────────┐    ┌──────────────┐    ┌─────────┐    ┌──────┐
│Cloud Scheduler│───▶│ Scheduler    │───▶│Firestore│───▶│ LINE │
│ (15日/月末)   │    │ Handler      │    │ (集計)  │    │通知  │
└───────────────┘    └──────────────┘    └─────────┘    └──────┘
```

### 2.3 月末精算・貯金計算の処理フロー（月末レポートに含む）

```
┌───────────────┐    ┌──────────────┐    ┌─────────┐    ┌──────┐
│Cloud Scheduler│───▶│ Scheduler    │───▶│Firestore│───▶│ LINE │
│ (月末)        │    │ Handler      │    │ (集計)  │    │通知  │
└───────────────┘    └──────────────┘    └─────────┘    └──────┘
```

**月末処理の詳細:**

1. **外食費用 - 貯金額の算出**
   - 各ユーザーの `diningBalance`（外食残金）を取得
   - 貯金額 = `diningBalance`（= 5万円 − 当月外食費用）
   - 各ユーザーの貯金額をレポートに表示

2. **買い物費用 - 精算額の算出**
   - 当月の買い物費用を各ユーザーごとに集計
   - 差額 = |ユーザーAの合計 − ユーザーBの合計|
   - 精算額 = 差額 ÷ 2
   - 少なく払った方 → 多く払った方に精算額を返金

### 2.4 月次残金リセットの処理フロー

```
┌───────────────┐    ┌──────────────┐    ┌─────────┐
│Cloud Scheduler│───▶│ Reset        │───▶│Firestore│
│ (毎月1日)     │    │ Handler      │    │ (更新)  │
└───────────────┘    └──────────────┘    └─────────┘
```

**詳細フロー:**

1. Cloud Scheduler が毎月1日に Reset Handler を呼び出し
2. 全ユーザーの `diningBalance` を 50000 に更新
3. `balanceResetAt` を現在時刻に更新

## 3. データ設計

### 3.1 Firestore コレクション構造

#### users コレクション（自動登録）
```javascript
{
  id: "LINE_USER_ID",           // LINE User ID（自動取得）
  displayName: "田中",           // LINE表示名（自動取得）
  groupId: "GROUP_ID",          // 所属グループID
  isActive: true,               // アクティブ状態
  diningBalance: 50000,         // 外食費用の残金（デフォルト: 5万円）
  balanceResetAt: Timestamp,    // 残金リセット日時（毎月1日にリセット）
  createdAt: Timestamp,         // 初回メッセージ送信時に自動作成
  updatedAt: Timestamp
}
```

**残金管理ルール（外食費用のみ対象）:**
- 各ユーザーは外食費用として月5万円の残金（`diningBalance`）を保有
- 外食費用の支出時のみ、支払ったユーザーの `diningBalance` から差し引く
- 買い物費用は `diningBalance` に影響しない
- 毎月1日に全ユーザーの `diningBalance` を50000にリセット
- **月末時点の `diningBalance` が各ユーザーの「貯金額」となる**

**自動登録の流れ:**
1. ユーザーが初めて画像を送信
2. LINE API から User ID と表示名を取得
3. users コレクションに自動登録
4. 以降は登録済みユーザーとして処理

#### expenses コレクション
```javascript
{
  id: "auto_generated_id",
  userId: "LINE_USER_ID",
  userName: "田中",
  amount: 1280,
  category: "外食費用",        // "外食費用" or "買い物費用"
  storeName: "サイゼリヤ",
  date: Timestamp,           // レシートの日付
  imageUrl: "gs://...",      // 画像保存先（オプション）
  calendarEventId: "xxx",    // Google Calendar イベントID
  createdAt: Timestamp
}
```

**カテゴリー別の処理:**
| カテゴリー | 残金への影響 | 月末処理 |
|------------|-------------|----------|
| 外食費用 | `diningBalance` から差し引く | 残金が貯金となる |
| 買い物費用 | 影響なし | 差額を精算 |

#### settings コレクション
```javascript
{
  id: "global",
  monthlyBudget: 50000,      // 月額予算（1人あたり）
  lineGroupId: "GROUP_ID",   // LINE グループID（初回メッセージ時に自動設定）
  calendarId: "xxx@group.calendar.google.com",
  firstHalfPayerId: "LINE_USER_ID_A",   // 前半担当者（1日〜15日）
  secondHalfPayerId: "LINE_USER_ID_B",  // 後半担当者（16日〜月末）
  updatedAt: Timestamp
}
```

**半月担当制（外食費用のみ）:**
- 1日〜15日: `firstHalfPayerId` のユーザーが外食の支払い担当
- 16日〜月末: `secondHalfPayerId` のユーザーが外食の支払い担当
- 担当者情報は定期レポートに表示される
- ※買い物費用には担当制は適用されない

### 3.2 Google Calendar イベント形式

```javascript
{
  summary: "[外食費用]  田中　￥1,280",
  description: "店舗: サイゼリヤ 渋谷店\n登録元: LINE家計簿Bot",
  start: {
    date: "2024-12-03"  // 終日イベント
  },
  end: {
    date: "2024-12-03"
  },
  colorId: "4"  // 外食費用=コーラルピンク(4), 買い物費用=シアン(7)
}
```

**カテゴリー判定:**
| カテゴリー | 判定基準 | カレンダー色 |
|------------|----------|-------------|
| 外食費用 | レストラン・飲食店 | コーラルピンク |
| 買い物費用 | スーパー・コンビニ | シアン |

## 5. セキュリティ設計

### 5.1 認証・認可

- LINE Webhook: 署名検証（X-Line-Signature）
- Google APIs: サービスアカウント認証
- Firestore: セキュリティルールで保護

### 5.2 シークレット管理

- Secret Manager で以下を管理:
  - LINE_CHANNEL_SECRET
  - LINE_CHANNEL_ACCESS_TOKEN
  - GEMINI_API_KEY
  - GOOGLE_CALENDAR_ID

## 6. エラーハンドリング

| エラー種別 | 対応 |
|------------|------|
| 画像解析失敗 | ユーザーに再送を依頼 |
| カレンダー登録失敗 | リトライ後、失敗通知 |
| DB保存失敗 | リトライ後、管理者に通知 |
| API制限超過 | キューイングで遅延処理 |

## 7. 監視・ログ

- Cloud Logging で全ログを収集
- Cloud Monitoring でアラート設定:
  - エラー率 > 5%
  - レイテンシ > 10秒
