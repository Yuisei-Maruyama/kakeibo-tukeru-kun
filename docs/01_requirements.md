# 要件定義書 - LINE × Google カレンダー 家計簿管理システム

## 1. プロジェクト概要

LINE と Google カレンダーを連携した家計簿管理システム。レシートや支払い画面のスクショを LINE で送信するだけで、自動的に支出を記録・管理できる。

## 2. 機能要件

### 2.1 画像解析・カレンダー登録機能

| 項目 | 内容 |
|------|------|
| トリガー | LINE トークルームへの画像送信 |
| 処理内容 | Gemini API で画像を解析し、支出情報を抽出 |
| 出力 | Google カレンダーに予定として登録 |

**カレンダー登録フォーマット:**
```
[[カテゴリー]]  [[支払い者名]]　[[金額(￥)]]
```

**例:**
```
[外食費用]  田中　￥1,280
[買い物費用]  鈴木　￥2,500
```

### 2.2 残高計算・即時返答機能

| 項目 | 内容 |
|------|------|
| トリガー | 画像送信と同時 |
| 処理内容 | 指定予算（例: 5万円）から支出を差し引き |
| 出力 | LINE トークルームに残高を返答 |

**返答例:**
```
✅ 登録しました！
📝 外食費用: ￥1,280（田中）
🏪 イタリアンレストラン
📅 2024/12/03

💰 今月の残高: ￥48,720
```

### 2.3 定期集計通知機能

| 項目 | 内容 |
|------|------|
| トリガー | 毎月15日と月末（自動実行） |
| 処理内容 | 前半月分の支出を集計 |
| 出力 | LINE トークルームに集計結果を通知 |

**通知タイミング:**
- 15日: 1日〜15日の集計
- 月末: 16日〜月末の集計

**通知例:**
```
📊 家計簿レポート（12/1〜12/15）

【支払い者別】
田中: ￥25,000
鈴木: ￥18,500

【カテゴリー別】
🍽️ 外食費用: ￥20,000
🛒 買い物費用: ￥23,500

💰 合計: ￥43,500
💵 残り予算: ￥6,500
```

## 3. 非機能要件

### 3.1 ユーザー数
- 支払い者: 2人

### 3.2 カテゴリー分類
- Gemini API による自動判定（支払い先から判断）
- カテゴリーは以下の **2種類のみ**:

| カテゴリー | 判定基準 | 例 |
|------------|----------|-----|
| **外食費用** | レストラン、飲食店、個人店での食事 | イタリアン、ラーメン屋、居酒屋、カフェ |
| **買い物費用** | スーパー、コンビニ、ドラッグストア等での購入 | イオン、セブンイレブン、マツキヨ |

**判定ロジック:**
- Gemini API がレシート/画面から店舗名を読み取り
- 店舗名から「外食」か「買い物」かを自動判定
- 判定できない場合は店舗の業態から推測

### 3.3 予算管理
- 月額予算: 設定可能（デフォルト: 5万円）
- 予算は設定変更可能とする

### 3.4 可用性
- 24時間365日稼働
- LINE Webhook の応答時間: 30秒以内

## 4. 制約条件

- LINE Messaging API の制限に準拠
- Google Calendar API のクォータに準拠
- Gemini API の利用制限に準拠

## 5. 用語定義

| 用語 | 説明 |
|------|------|
| 支払い者 | 画像を LINE に送信したユーザー |
| カテゴリー | 支出の分類（外食費用 or 買い物費用） |
| 外食費用 | レストラン・飲食店での支払い |
| 買い物費用 | スーパー・コンビニ等での支払い |
| 予算 | 月間の支出上限額 |
| 半月 | 1日〜15日 または 16日〜月末 |
