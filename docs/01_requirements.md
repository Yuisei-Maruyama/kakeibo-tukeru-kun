# 要件定義書 - LINE × Google カレンダー 家計簿管理システム

## 1. プロジェクト概要

LINE と Google カレンダーを連携した家計簿管理システム。レシートや支払い画面のスクショを LINE で送信するだけで、自動的に支出を記録・管理できる。

---

## 🌏 タイムゾーン仕様（重要）

**本システムは日本時間（JST、UTC+9）で動作することを保証します。**

### タイムゾーン設計原則

| 項目 | 仕様 |
|------|------|
| **基準タイムゾーン** | JST（日本標準時、UTC+9） |
| **日付境界** | JST 00:00:00 〜 23:59:59 |
| **月の定義** | JST基準での1日〜月末 |
| **Cloud Functions実行環境** | UTC（内部でJSTに変換） |

### 実装ルール（開発者向け）

```
⚠️ Cloud FunctionsはUTC環境で動作するため、以下のルールを厳守すること：

1. Date オブジェクトの作成
   ✅ new Date(Date.UTC(year, month, day, ...))
   ❌ new Date(year, month, day, ...)

2. Date から年月日を取得
   ✅ date.getUTCFullYear(), date.getUTCMonth(), date.getUTCDate()
   ❌ date.getFullYear(), date.getMonth(), date.getDate()

3. 日付の設定
   ✅ Date.UTC() で新しい Date を作成
   ❌ setHours(), setDate() などのローカルメソッド

4. JST現在時刻の取得
   ✅ utils/date.ts の getJSTDate() を使用
   ❌ new Date() を直接使用

5. 日付文字列のフォーマット
   ✅ utils/date.ts の formatDateYYYYMMDD() を使用
   ✅ date.getUTCFullYear() + date.getUTCMonth() + date.getUTCDate()
   ❌ date.getFullYear() + date.getMonth() + date.getDate()
```

### 影響範囲

- 支出の日付判定（どの月に属するか）
- 残高の更新（現在月の支出のみ）
- 定期レポートの期間計算
- サブスクリプションの配送日計算
- カレンダー同期の日付範囲

## 2. 機能要件

### 2.1 画像解析・カレンダー登録機能

| 項目 | 内容 |
|------|------|
| トリガー | LINE トークルームへの画像送信 |
| 処理内容 | Gemini API で画像を解析し、支出情報を抽出 |
| 出力 | Google カレンダーに予定として登録 |

**カレンダー登録フォーマット:**
```
[[カテゴリー]]  [[支払い者名]]　[[金額(￥)]]
```

**例:**
```
[外食費用]  田中　￥1,280
[買い物費用]  鈴木　￥2,500
```

### 2.2 残高計算・即時返答機能

| 項目 | 内容 |
|------|------|
| トリガー | 画像送信と同時 |
| 処理内容 | カテゴリーに応じた残高計算 |
| 出力 | LINE トークルームに残高を返答 |

**返答例（外食費用）:**
```
✅ 登録しました！
📝 外食費用: ￥1,280（田中）
🏪 イタリアンレストラン
📅 2024/12/03

💰 田中の外食残高: ￥48,720
```

**返答例（買い物費用）:**
```
✅ 登録しました！
📝 買い物費用: ￥2,500（鈴木）
🏪 イオン
📅 2024/12/03

🛒 鈴木の今月の買い物合計: ￥15,500
```

### 2.3 定期集計通知機能

| 項目 | 内容 |
|------|------|
| トリガー | 毎月15日と月末（自動実行） |
| 処理内容 | 期間内の支出を集計し、精算・貯金を計算 |
| 出力 | LINE トークルームに集計結果を通知 |

**通知タイミング:**
- **15日（中間レポート）**: 1日〜15日の半月集計
- **月末（最終レポート）**: 16日〜月末の半月集計 + **月間全体（1日〜月末）の精算・貯金サマリー**

**レポート構成の詳細:**

| レポート | 対象期間 | 外食費用 | 買い物費用 | 外食残高 | 月間サマリー |
|---------|---------|---------|-----------|---------|------------|
| 15日 | 1日〜15日 | ✅ 半月分 | ✅ 半月分 | ✅ 現在残高 | ❌ なし |
| 月末 | 16日〜月末 | ✅ 半月分 | ✅ 半月分 | ✅ 現在残高 | ✅ **月全体（1日〜月末）**の貯金・精算 |

**注意:**
- 月間サマリーの貯金額・精算額は、**15日レポート + 月末レポート = 月全体**の計算結果です
- 例：外食費用の貯金 = 50,000円 - (1日〜15日の使用 + 16日〜月末の使用)
- **外食担当者は`@初期設定`または`@設定変更`コマンドで設定**されます（設定がない場合は表示されません）

### 2.4.1 対話形式入力機能

**概要:**
`@追加`、`@予定`、または `@削除` のみを送信すると、対話形式で必要な情報を順番に入力できる。

**対話フロー（@追加の場合）:**
```
ユーザー: @追加
Bot: 📝 支出を登録します

カテゴリーを選択してください
1️⃣ 外食費用
2️⃣ 買い物費用
3️⃣ 旅行費用

ユーザー: 1
Bot: 支払い者名を入力してください
（@自分 で自分の名前、@メンションでユーザー指定）

ユーザー: @自分
Bot: 金額を入力してください（数字のみ）

ユーザー: 1280
Bot: 日付を入力してください
（例: 12/15、2024/12/15）
日付形式:
- M/D: 今年の日付（例: 5/22 → 2024/5/22）
- YYYY/M/D: 年を指定（例: 2024/5/22）
「今日」と入力すると今日の日付になります

ユーザー: 今日
Bot: ✅ 登録しました！
📝 外食費用: ￥1,280（鈴木）

### 2.4.2 年月指定機能

**概要:**
`@追加`、`@予定`、`@削除`、`@集計`、`@履歴` コマンドで年月を指定できる。

#### 日付形式の指定方法

**対応形式:**

| 形式 | 説明 | 例 | 解釈 |
|------|------|-----|------|
| `M/D` | 今年の日付 | `5/22` | 2024年5月22日 |
| `YYYY/M/D` | 年を指定 | `2024/5/22` | 2024年5月22日 |
| `YYYY-M-D` | 年を指定（ハイフン） | `2024-5-22` | 2024年5月22日 |
| `今日` | 今日の日付 | `今日` | 実行日の日付 |

**コマンド別の使用例:**

```
@追加 田中 外食費用 1280              # 今日
@追加 田中 外食費用 1280 12/15        # 今年の12月15日
@追加 田中 外食費用 1280 2024/12/15   # 2024年12月15日
@追加 田中 外食費用 1280 2023/5/22    # 2023年5月22日（過去）

@予定 会議                            # 今日
@予定 会議 12/15                      # 今年の12月15日
@予定 会議 2024/12/15                 # 2024年12月15日
@予定 会議 12/15 14:30                # 開始時刻のみ（終了時刻は自動的に+1時間）
@予定 会議 12/15 14:30 16:00          # 開始・終了時刻を指定

@削除 @自分 外食費用 1280             # 今日
@削除 @自分 外食費用 1280 12/15       # 今年の12月15日
@削除 @自分 外食費用 1280 2024/5/22   # 2024年5月22日
```

#### 年月指定での集計・履歴表示

**対応形式:**

| 形式 | 説明 | 例 | 解釈 |
|------|------|-----|------|
| 指定なし | 今月全体 | `@集計` | 今月1日〜月末 |

| `YYYY/M` | 年月を指定 | `@集計 2024/12` | 2024年12月全体 |
| `YYYY-M` | 年月を指定（ハイフン） | `@集計 2024-12` | 2024年12月全体 |

### 2.5 サブスク（定期支払い）自動登録機能

| 項目 | 内容 |
|------|------|
| トリガー | 毎月1日（自動実行） |
| 処理内容 | 登録されているサブスクをその月の該当日付で自動登録 |
| 出力 | LINE トークルームに登録結果を通知 |

**サブスクとは:**
- Netflix、Spotify などの定期課金サービス
- Amazon定期おトク便などの定期購買
- 定期的に同じ金額が発生する支出

**登録項目:**
- **支払い者**: 誰が支払うか（`@自分`またはメンションで指定）
- **支払い内容**: サービス名や定期購買の内容（例: Netflix、Amazon定期便 猫砂）
- **金額**: 支払い金額
- **開始日**: 初回の支払日（例: 12/15、2024/12/15）
- **間隔単位**: 週ごと or 月ごと
- **間隔数値**: 何週間/何ヶ月ごとか（例: 1=毎週/毎月、2=2週間/2ヶ月ごと）

**間隔の例:**
| 設定 | 意味 |
|------|------|
| 週ごと・1 | 毎週 |
| 週ごと・2 | 2週間に1回 |
| 週ごと・4 | 4週間に1回 |
| 月ごと・1 | 毎月 |
| 月ごと・2 | 2ヶ月に1回 |
| 月ごと・3 | 3ヶ月に1回 |

**自動登録の動作:**
1. 毎月1日の朝に自動実行
2. 登録されているサブスクを全て取得
3. 各サブスクの開始日と間隔から、その月に該当する日付を算出
4. 該当する日付がある場合、「買い物費用」として登録
5. Googleカレンダーにも予定として登録
6. LINEに登録完了通知を送信

**日付算出の例:**
- **開始日: 12/10、2週間ごと** → 12/10, 12/24, 1/7, 1/21, ... と計算
  - 1月分を処理する際、1/7 と 1/21 が該当
- **開始日: 12/15、毎月** → 12/15, 1/15, 2/15, ... と計算
  - 1月分を処理する際、1/15 が該当
- **開始日: 11/20、2ヶ月ごと** → 11/20, 1/20, 3/20, ... と計算
  - 1月分を処理する際、1/20 が該当
  - 12月分を処理する際は該当なし

**通知例:**
```
🔄 サブスク自動登録完了
━━━━━━━━━━━━━━━

2024年12月分のサブスクを登録しました

✅ Netflix（鈴木・¥1,490・12/15）
✅ Amazon定期便 猫砂（田中・¥1,980・12/10）
✅ Amazon定期便 猫砂（田中・¥1,980・12/24）
```

**注意事項:**
- サブスクは全て「買い物費用」として登録される
- 外食費用の残高には影響しない
- 週単位の場合、同じ月に複数回該当する場合がある（2週間ごとなら最大3回）
- 月単位で31日を開始日にした場合、日数が少ない月は月末日に調整される
- 削除しても過去に自動登録された支出は削除されない（@削除で個別に削除が必要）

### 2.5.1 家賃自動登録機能

**自動登録の動作:**
1. 毎月1日の朝に自動実行
2. 登録されている家賃情報を取得
3. その月の月末日を算出
4. 月末日に「家賃費用」としてGoogleカレンダーに登録

━━━━━━━━━━━━━━━

2024年12月分の家賃を登録しました

✅ 家賃費用（鈴木・¥120,000・12/31）
```

**注意事項:**
- 家賃は「家賃費用」として独立したカテゴリーで管理
- 外食費用の残高には影響しない
- 買い物費用の精算対象外
- 家賃情報が未登録の場合はスキップ（通知なし）

### 2.6 Googleカレンダー同期機能

| 項目 | 内容 |
|------|------|
| トリガー | 毎日深夜3:00（自動実行） |
| 処理内容 | Googleカレンダーの当月の支出イベントをFirestoreに同期 |
| 出力 | なし（バックグラウンド処理） |

**同期ロジック:**
1. Googleカレンダーから当月のイベントを取得
2. 各イベントのタイトルから以下の情報をパース：
   - **カテゴリー**: 部分一致で自動判定（柔軟対応）
     - 外食費用: 「外食」「外食費」を含む
     - 買い物費用: 「買い物」「買物」を含む
   - **ユーザー名**: あいまい検索（部分一致）
   - **金額**: 数字（カンマ・円記号・円文字の有無は問わない）
   - **店舗名**: `()`付きの場合のみ（オプション）
3. カレンダーイベントIDをキーとして、Firestoreに既に存在するかチェック
4. 存在しない場合のみ、Firestoreの`expenses`コレクションに追加
5. 外食費用の場合、ユーザーの残高を更新
6. **カレンダーで削除されたイベントを検出し、Firestoreからも自動削除**
   - Firestoreから当月の支出を取得
   - カレンダーイベントIDを持つがカレンダーに存在しない支出を削除
   - 外食費用の場合、削除時に残高を戻す

**タイトル形式（柔軟なパース対応）:**

以下のいずれの形式でも認識可能：

| 形式 | 例 |
|------|-----|
| `[カテゴリー] ユーザー名 ¥金額 (店舗名)` | `[外食費用] 田中 ¥1,280 (イタリアン)` |
| `[カテゴリー] ユーザー名 金額 (店舗名)` | `[外食費用] 田中 1280 (イタリアン)` |
| `[カテゴリー] ユーザー名 金額円 (店舗名)` | `[外食費用] 田中 1280円 (イタリアン)` |
| `カテゴリー ユーザー名 ¥金額 (店舗名)` | `外食費用 田中 ¥1,280 (イタリアン)` |
| `カテゴリー ユーザー名 金額 (店舗名)` | `外食費用 田中 1280 (イタリアン)` |
| `カテゴリー ユーザー名 金額円 (店舗名)` | `外食費用 田中 1280円 (イタリアン)` |
| **部分一致（外食）** | `外食 田中 1280円` |
| **部分一致（買い物）** | `買い物 鈴木 2500` |

**柔軟な要素:**
- ✅ `[]` の有無は問わない
- ✅ `¥` の有無は問わない
- ✅ `円` の有無は問わない
- ✅ 金額の `,`（カンマ）は自動削除
- ✅ 店舗名 `()` はオプション（省略可能）
- ✅ **カテゴリーは部分一致で自動判定**
  - 外食費用: 「外食」「外食費」
  - 買い物費用: 「買い物」「買物」
- ✅ ユーザー名はあいまい検索（部分一致）に対応

**パース例:**

| カレンダータイトル | 解析結果 |
|-------------------|---------|
| `[外食費用] 田中 ¥1,280` | カテゴリー: 外食費用、ユーザー名: 田中、金額: 1280、店舗名: 手動追加 |
| `外食費用 田中 1280` | カテゴリー: 外食費用、ユーザー名: 田中、金額: 1280、店舗名: 手動追加 |
| `外食費用 田中 1280円` | カテゴリー: 外食費用、ユーザー名: 田中、金額: 1280、店舗名: 手動追加 |
| `[買い物費用] 鈴木 ¥2,500` | カテゴリー: 買い物費用、ユーザー名: 鈴木、金額: 2500、店舗名: 手動追加 |
| `買い物費用 鈴木 2500円` | カテゴリー: 買い物費用、ユーザー名: 鈴木、金額: 2500、店舗名: 手動追加 |
| `[外食費用] 田中 ¥1,280 (イタリアン)` | カテゴリー: 外食費用、ユーザー名: 田中、金額: 1280、店舗名: イタリアン |
| `外食費用 田中太 1280円` | カテゴリー: 外食費用、ユーザー名: 田中太郎（あいまい検索でマッチ）、金額: 1280、店舗名: 手動追加 |

**ユーザー名のあいまい検索:**

ユーザー名が完全一致しない場合、以下のロジックで検索：
1. **完全一致**: 入力された名前と登録名が完全一致（優先）
2. **部分一致（含む）**: 入力が登録名に含まれる
   - 例: 入力「田中」→ 登録「田中太郎」にマッチ
3. **部分一致（含まれる）**: 登録名が入力に含まれる
   - 例: 入力「田中太郎さん」→ 登録「田中」にマッチ
4. **正規化**: 空白除去・小文字化して比較

**注意事項:**
- カレンダーイベントの色（colorId: 4 or 7）は問わない（タイトル形式のみで判定）
- パース失敗したイベントはスキップ（ログに記録）
- 同じカレンダーイベントIDの重複追加を防止
- 手動でカレンダーに追加された支出も自動的にFirestoreに反映される
- 手動でカレンダーから削除された支出も自動的にFirestoreから削除される
- ユーザー名が見つからない場合は、そのイベントをスキップ（エラーカウント）

## 3. 非機能要件

### 3.1 ユーザー数
- 支払い者: 2人

### 3.2 カテゴリー分類
- Gemini API による自動判定（支払い先から判断）
- カテゴリーは以下の **3種類**:

| カテゴリー | 判定基準 | 例 |
|------------|----------|-----|
| **外食費用** | レストラン、飲食店、個人店での食事 | イタリアン、ラーメン屋、居酒屋、カフェ |
| **買い物費用** | スーパー、コンビニ、ドラッグストア等での購入 | イオン、セブンイレブン、マツキヨ |
| **旅行費用** | 交通費、宿泊費、観光施設、旅行先での支出 | 新幹線、飛行機、ホテル、お土産 |

**判定ロジック:**
- Gemini API がレシート/画面から店舗名を読み取り
- 店舗名から「外食費用」「買い物費用」「旅行費用」を自動判定
- 判定できない場合は店舗の業態から推測

**カレンダー色分け:**

| カテゴリー/種別 | 色 | colorId |
|----------------|-----|---------|
| 外食費用 | コーラルピンク | 4 |
| 買い物費用 | シアン | 7 |
| 旅行費用 | ラベンダー（紫） | 1 |
| 家賃費用 | バナナ（黄色） | 5 |
| 予定 | バジル（緑） | 10 |

### 3.3 予算管理

カテゴリーによって予算管理方法が異なる。

---

#### 🍽️ 外食費用の管理

##### 個別残金制度
- **各ユーザーが月5万円の残金を保有**
- 残金は**ユーザーごとに独立**して管理される
- 外食費用の支出は**支払いを行ったユーザーの残金から差し引かれる**
- **残高の更新は現在の月の支出のみに適用**
  - 過去の月の支出を登録しても、現在の月の残高には影響しない
  - 例: 2月に1月の支出を登録した場合、支出は1月の集計に含まれるが、2月の残高は変わらない

##### 半月担当制
- **1日〜15日**: ユーザーAが支払い担当
- **16日〜月末**: ユーザーBが支払い担当
- 担当期間中は、担当者が優先的に支払いを行う運用
- ※担当外のユーザーが支払った場合も、そのユーザーの残金から差し引かれる
- **担当者の設定方法**: `@初期設定`または`@設定変更`コマンドで対話形式で設定
  - 前半担当者（1日〜15日）と後半担当者（16日〜月末）をそれぞれ選択
  - 設定後は定期レポートに「👤 外食担当: ○○」として表示される
  - 設定がない場合、レポートに外食担当者は表示されない

##### 月次リセット
- **毎月1日に全ユーザーの残金が5万円にリセット**される
- リセットは自動で行われる

##### 貯金計算（月末）
- **月末に各ユーザーの残金が「貯金」となる**
- 貯金額 = 5万円 − 当月の外食費用支出
- 月末レポートで各ユーザーの貯金額を通知

##### 予算設定
- デフォルト予算: 5万円/人/月
- 予算は設定変更可能とする

---

#### 🛒 買い物費用の管理

##### 各自負担・月末精算制
- 買い物費用は**残金制度の対象外**
- 各ユーザーが支払った買い物費用を**月末に集計**
- **差額を精算**して、多く払った方に返金

##### 精算計算（月末）
- 各ユーザーの当月買い物費用合計を算出
- 差額 = |ユーザーAの合計 − ユーザーBの合計|
- **少なく払った方 → 多く払った方** に差額の半分を返金

**精算例:**
```
田中の買い物費用: ￥30,000
鈴木の買い物費用: ￥20,000
差額: ￥10,000

→ 鈴木が田中に ￥5,000 を返金
```

---

#### 🧳 旅行費用の管理

##### 各自負担・月末精算制
- 旅行費用は**残金制度の対象外**（外食残高には影響しない）
- 買い物費用と同様に、各ユーザーが支払った旅行費用を**月末に集計**
- **差額を精算**して、多く払った方に返金

##### 精算計算（月末）
- 各ユーザーの当月旅行費用合計を算出
- 差額 = |ユーザーAの合計 − ユーザーBの合計|
- **少なく払った方 → 多く払った方** に差額の半分を返金

**精算例:**
```
田中の旅行費用: ￥35,000
鈴木の旅行費用: ￥28,000
差額: ￥7,000

→ 鈴木が田中に ￥3,500 を返金
```

##### 旅行費用の対象
- 交通費: 新幹線、飛行機、バス、タクシー、レンタカー等
- 宿泊費: ホテル、旅館、民宿、Airbnb等
- 観光施設: 入場料、レジャー施設利用料等
- お土産: 旅行先でのお土産購入
- その他: 旅行に関連する支出全般

### 3.4 可用性
- 24時間365日稼働
- LINE Webhook の応答時間: 30秒以内

## 6. 用語定義

| 用語 | 説明 |
|------|------|
| 支払い者 | 画像を LINE に送信したユーザー |
| カテゴリー | 支出の分類（外食費用 / 買い物費用 / 旅行費用 / 家賃費用） |
| 外食費用 | レストラン・飲食店での支払い（残金制度対象） |
| 買い物費用 | スーパー・コンビニ等での支払い（月末精算対象） |
| 旅行費用 | 交通費・宿泊費・観光施設等の支払い（月末精算対象） |
| 家賃費用 | 毎月発生する固定家賃（精算対象外、月末自動登録） |
| 予算 | 月間の支出上限額 |
| 残金 | 各ユーザーが保有する外食費用の月間予算残高（5万円/月） |
| 貯金 | 月末時点の外食残金（5万円 − 当月外食費用） |
| 精算 | 買い物費用の差額を月末に調整する処理 |
| 半月 | 1日〜15日 または 16日〜月末 |
| 担当者 | 半月ごとに切り替わる外食の支払い担当ユーザー |
| 月次リセット | 毎月1日に外食残金を初期値（5万円）に戻す処理 |
