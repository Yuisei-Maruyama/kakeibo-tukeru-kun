# 要件定義書 - LINE × Google カレンダー 家計簿管理システム

## 1. プロジェクト概要

LINE と Google カレンダーを連携した家計簿管理システム。レシートや支払い画面のスクショを LINE で送信するだけで、自動的に支出を記録・管理できる。

## 2. 機能要件

### 2.1 画像解析・カレンダー登録機能

| 項目 | 内容 |
|------|------|
| トリガー | LINE トークルームへの画像送信 |
| 処理内容 | Gemini API で画像を解析し、支出情報を抽出 |
| 出力 | Google カレンダーに予定として登録 |

**カレンダー登録フォーマット:**
```
[[カテゴリー]]  [[支払い者名]]　[[金額(￥)]]
```

**例:**
```
[外食費用]  田中　￥1,280
[買い物費用]  鈴木　￥2,500
```

### 2.2 残高計算・即時返答機能

| 項目 | 内容 |
|------|------|
| トリガー | 画像送信と同時 |
| 処理内容 | カテゴリーに応じた残高計算 |
| 出力 | LINE トークルームに残高を返答 |

**返答例（外食費用）:**
```
✅ 登録しました！
📝 外食費用: ￥1,280（田中）
🏪 イタリアンレストラン
📅 2024/12/03

💰 田中の外食残高: ￥48,720
```

**返答例（買い物費用）:**
```
✅ 登録しました！
📝 買い物費用: ￥2,500（鈴木）
🏪 イオン
📅 2024/12/03

🛒 鈴木の今月の買い物合計: ￥15,500
```

### 2.3 定期集計通知機能

| 項目 | 内容 |
|------|------|
| トリガー | 毎月15日と月末（自動実行） |
| 処理内容 | 期間内の支出を集計し、精算・貯金を計算 |
| 出力 | LINE トークルームに集計結果を通知 |

**通知タイミング:**
- 15日: 1日〜15日の集計（中間レポート）
- 月末: 16日〜月末の集計 + **月間精算・貯金レポート**

## 3. 非機能要件

### 3.1 ユーザー数
- 支払い者: 2人

### 3.2 カテゴリー分類
- Gemini API による自動判定（支払い先から判断）
- カテゴリーは以下の **2種類のみ**:

| カテゴリー | 判定基準 | 例 |
|------------|----------|-----|
| **外食費用** | レストラン、飲食店、個人店での食事 | イタリアン、ラーメン屋、居酒屋、カフェ |
| **買い物費用** | スーパー、コンビニ、ドラッグストア等での購入 | イオン、セブンイレブン、マツキヨ |

**判定ロジック:**
- Gemini API がレシート/画面から店舗名を読み取り
- 店舗名から「外食費用」か「買い物費用」かを自動判定
- 判定できない場合は店舗の業態から推測

### 3.3 予算管理

カテゴリーによって予算管理方法が異なる。

---

#### 🍽️ 外食費用の管理

##### 個別残金制度
- **各ユーザーが月5万円の残金を保有**
- 残金は**ユーザーごとに独立**して管理される
- 外食費用の支出は**支払いを行ったユーザーの残金から差し引かれる**

##### 半月担当制
- **1日〜15日**: ユーザーAが支払い担当
- **16日〜月末**: ユーザーBが支払い担当
- 担当期間中は、担当者が優先的に支払いを行う運用
- ※担当外のユーザーが支払った場合も、そのユーザーの残金から差し引かれる

##### 月次リセット
- **毎月1日に全ユーザーの残金が5万円にリセット**される
- リセットは自動で行われる

##### 貯金計算（月末）
- **月末に各ユーザーの残金が「貯金」となる**
- 貯金額 = 5万円 − 当月の外食費用支出
- 月末レポートで各ユーザーの貯金額を通知

##### 予算設定
- デフォルト予算: 5万円/人/月
- 予算は設定変更可能とする

---

#### 🛒 買い物費用の管理

##### 各自負担・月末精算制
- 買い物費用は**残金制度の対象外**
- 各ユーザーが支払った買い物費用を**月末に集計**
- **差額を精算**して、多く払った方に返金

##### 精算計算（月末）
- 各ユーザーの当月買い物費用合計を算出
- 差額 = |ユーザーAの合計 − ユーザーBの合計|
- **少なく払った方 → 多く払った方** に差額の半分を返金

**精算例:**
```
田中の買い物費用: ￥30,000
鈴木の買い物費用: ￥20,000
差額: ￥10,000

→ 鈴木が田中に ￥5,000 を返金
```

### 3.4 可用性
- 24時間365日稼働
- LINE Webhook の応答時間: 30秒以内

## 4. 制約条件

- LINE Messaging API の制限に準拠
- Google Calendar API のクォータに準拠
- Gemini API の利用制限に準拠

## 5. 用語定義

| 用語 | 説明 |
|------|------|
| 支払い者 | 画像を LINE に送信したユーザー |
| カテゴリー | 支出の分類（外食費用 or 買い物費用） |
| 外食費用 | レストラン・飲食店での支払い（残金制度対象） |
| 買い物費用 | スーパー・コンビニ等での支払い（月末精算対象） |
| 予算 | 月間の支出上限額 |
| 残金 | 各ユーザーが保有する外食費用の月間予算残高（5万円/月） |
| 貯金 | 月末時点の外食残金（5万円 − 当月外食費用） |
| 精算 | 買い物費用の差額を月末に調整する処理 |
| 半月 | 1日〜15日 または 16日〜月末 |
| 担当者 | 半月ごとに切り替わる外食の支払い担当ユーザー |
| 月次リセット | 毎月1日に外食残金を初期値（5万円）に戻す処理 |
